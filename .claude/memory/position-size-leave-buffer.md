开仓算仓位时**不要把 max_loss 打满**，留 buffer 覆盖"预估成交价偏离参考价"导致的止损距变大（2026-07-17 用户立）。

**为什么**：信号参考价（发信号时）≠ 实际成交价（响铃时刻）。做空若成交价 < 参考价，止损距 = 止损−成交价 变大，按参考价算满的仓位会超 max_loss。2026-07-17 午市第二笔：参考价 1145/止损 1170/按 10000÷25=4 手打满，实际成交 1136/止损距 34/损失 400×34=13600 > max_loss 10000，被迫发减仓/调止损信号补救（后处理）。留 buffer 能避免这种后处理。

**怎么用（两选一，按标的波动选）**：

1. **按参考价 + buffer（简化，推荐日常用）**：
   - 仓位 = max_loss × (1−buffer) ÷ (止损−参考价)，buffer 一般 **20-30%**（高波动标的放大）。
   - 即 max_loss 只用到 7000-8000，留 2000-3000 给成交价偏离。
   - 偏离在 buffer 内 → 不用后处理；偏离超 buffer 才减仓。

2. **按价格范围下沿算（最严谨，保守）**：
   - 做空范围 = [(止盈+止损)/2, 止损]，按下沿（最差成交价、盈亏比=1 价）算 → 止损距 = (止损−止盈)/2，仓位 = max_loss ÷ ((止损−止盈)/2)。
   - 保证整个范围内任意成交价损失 ≤ max_loss。代价：仓位较小（约为参考价算的一半）。

**层级（重要，2026-07-17 用户补）**——buffer 优先、减仓兜底，能不动就不动：
1. **第一道：开仓留 buffer**（上面方案 A/B）。响铃后按实际成交价算损失，若 ≤ max_loss（buffer 够覆盖偏离）→ **不用动，正常持有**。
2. **第二道：减仓补救（仅当偏离超 buffer）**。只有按实际成交价算损失仍 > max_loss 时，才减仓——按实际成交价 + max_loss 重算合规仓位、减到不超（如 2 手→1 手）。

**配套**（减小偏离本身）：发信号前 snapshot 最新价作参考价（不用过时的 monitor 价）；响铃后立即 monitor 取响铃时刻成交价验证（见 [[verify-signal-price-after-send]]）。

**与 max_loss 关系**：max_loss 是硬上限（config 10000），buffer 是仓位计算时的预留——让"成交价偏离"这个不确定性有吸收空间，不溢出到事后补救。
