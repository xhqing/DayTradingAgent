默认用户执行了 AI 发的所有信号（开仓/加仓/移动止损/移动止盈/平仓），AI 按"假设执行"管理，**不每次问用户"挂了吗/触发了吗/成交了吗"**（2026-07-17 用户立）。

**为什么**：用户明确"默认都有执行"。反复问确认是多余的反 confirm，打断盯盘节奏（今天我曾反复问"1145 止损挂了吗/触发了吗"，被用户纠正"默认都有执行"）。

**怎么用**：
- 发信号 → 响铃后 AI 自己取价验证（在范围内 = 成交）→ 按假设管理（假设用户执行了），不问用户。
- 移动止损/止盈建议发后 → 默认用户挂了 → 价格到触发位 = 触发 → 按被动平仓标记（不发 🔴 不响铃，只标记 + signals 记录）。
- 用户若实际没执行/异常，会主动告知 → 那时才据实调整。

与 [[verify-signal-price-after-send]]（响铃后取价判范围）配合：验证是 AI 自己取价判（不问用户），判完按假设管理。与 [[signal-is-real-trading]] 一致：信号=真实交易，假设执行、不查 positions、不等反馈。
