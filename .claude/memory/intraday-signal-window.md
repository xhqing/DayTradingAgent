发信号与盯盘时段规则（2026-07-16 立、2026-07-19 大改）。强约束规范在 trade skill（SKILL.md 护栏 10「发信号与盯盘时段」、「时点平仓与停止盯盘约束」章节、信号模式总则第 2 条 + 第 5 条复盘），本条记决策与原因、作跨会话背景。

**规则要点（2026-07-19 最新）**：
- **五类信号（开仓 🟢 / 加仓 🔵 / 减仓 🟠 / 平仓 🔴 / 移动止损 🟡）只在盘中发**，盘前 / 盘后 / 夜盘不发。盯盘也只在盘中。
- 港股：盘中 09:30-12:00 / 13:00-16:00。两个强制终止边界：**12:00 午休前**（主动平仓 + 终止盯盘，13:00 午市不自动恢复、等用户指令）、**16:00 收盘前**（主动平仓 + 终止盯盘）。
- 美股：盘中**美东 09:30-16:00**（北京夏令时 21:30-次日 04:00 / 冬令时 22:30-次日 05:00）。**盯到用户发出停止盯盘指令、或美东 16:00 收盘**（取先到者）——用户大概率北京晚上睡觉前喊停。**撤销原「美东 12:00 固定停盯」**。
- **撤销留单策略（2026-07-19 立）**：无论哪个时点停止盯盘，**停前一律主动平仓、不留单**（原「留单靠条件单兜底赌午盘」整条作废）。
- **盯盘不自主启动**：收到用户盯盘指令才开始；用户若在盘前发指令，只做预热（preflight / 方向研判 / hot_list）不发信号，开盘后才发。
- **复盘由用户指令触发、盘中不复盘**；记录信号（append signals/）不是复盘，照常做。

**2026-07-19 改动背景**（撤销 2026-07-16 的几条）：
- **美股盯到收盘**：原 2026-07-16「美东 12:00 固定停盯（因 12:00 ET = 北京 00:00 用户睡觉）」改为「盯到用户喊停或 16:00 收盘」——用户决定不再固定 12:00 停，而是自己（大概率睡觉前）喊停、少数盯到收盘。
- **撤销留单**：原「12:00 停盯前 主动平 vs 留单二选一」整条作废，停前一律主动平。
- **港股收盘顺延**：原 15:45（长桥日内融强平时点）顺延到 16:00（市场收盘）——因账户改主账户、不再有 15:45 强平（见 [[longbridge-main-account]]）。
- 2026-07-16 的「撤销 24h 盯盘」「复盘用户触发」仍保留。

**How to apply**：盯盘 cron 只在用户指令后建（关联 [[cron-replace-delete-old]]）；preflight.py 美股时段已同步为「美东 09:30-16:00 可发信号」；盘前 / 非窗口时段不输出五类信号、不自主复盘；停盯前必主动平、不留单。信号模式本身不变（AI 发信号、用户执行）。
