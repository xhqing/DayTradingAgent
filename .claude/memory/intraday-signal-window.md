2026-07-16 用户修订 DayTradingAgent 的发信号与盯盘时段规则，撤销 2026-07-15 的「美股 24 小时盯盘」。强约束规范已写入 trade skill（SKILL.md 护栏 10「发信号与盯盘时段」、「时点平仓与停止盯盘约束」章节、自演练模式第 5 条复盘），本条记决策与原因、作跨会话背景。

**规则要点**：
- **四类信号（开仓 🟢 / 平仓 🔴 / 移动止损 🟡 / 移动止盈 🟠）只在盘中发**，盘前 / 盘后 / 夜盘不发。盯盘也只在盘中。
- 港股：盘中 09:30-12:00 / 13:00-16:00。
- 美股：盘中且**仅美东 12:00 之前**——信号窗口 = 美东 09:30-12:00（北京夏令时 21:30-24:00 / 冬令时 22:30-次日 01:00）；美东 12:00 后**停止盯盘**（不再「用户睡觉盯到美股收盘」）。
- **盯盘不自主启动**：收到用户盯盘指令才开始；用户若在盘前发指令，只做预热（preflight / 方向研判 / hot_list）不发信号，开盘后才发。
- **三个强制终止盯盘边界**（到点终止、不拖）：港股 12:00 午休前（平仓 + 终止盯盘，13:00 午市**不自动恢复**、等用户指令）、港股 15:45 收盘前（平仓 + 终止盯盘）、美股美东 12:00 前（终止盯盘 + 平仓决策：主动平 / 留单靠止损止盈单被动平）。三个边界都是终止、非暂停，停后须等用户盯盘指令才重启。
- **复盘由用户指令触发、盘中不复盘**（覆盖 2026-07-15「自主复盘」）；记录信号（append signals/）不是复盘，照常做。

**Why**：① 美东 12:00 = 北京时间 00:00（夏令时），用户要睡觉，12:00 后盯盘发信号无意义、靠 App 条件单兜底即可；② 撤销 24 小时盯盘减负（夜里不空转、省 token / API）；③ 盘中复盘会打断盯盘节奏、影响信号时效性，故复盘挪到用户指令触发。

**How to apply**：盯盘 cron 只在用户指令后建、撞三边界即停（关联 [[cron-replace-delete-old]]）；preflight.py 的美股时段判断已同步改为「仅美东 09:30-12:00 可发信号」；盘前 / 非窗口时段不输出四类信号、不自主复盘。信号模式本身不变（AI 发信号、用户执行，见 [[longbridge-cli-revoked-signal-mode]]）。
