发完开仓 🟢 / 加仓 🔵 信号后，**必须立即再采一次数据验证信号参考价是否还能成交**（2026-07-17 用户立）。

**为什么**：从发出信号到用户在 App 挂单成交有时间差（AI 写信号 + 用户读信号 + 用户 App 操作，几十秒到几分钟），期间价格在变。做空限价卖参考价 X，若现价已跌到 X 以下，限价卖单不会立即成交（需价格回到 X 才撮合）；做多同理反向。不验证就以为"加仓成功了"，实际可能没成交，导致后续仓位管理（浮盈、止损、加仓基础）算错。

**怎么用**：发完开仓/加仓信号 → 立即 monitor / snapshot 取最新价 → 对比信号参考价：
- 价格偏离小（参考价仍可成交）：确认成交，继续盯。
- 价格偏离大（参考价不可成交）：告知用户偏离多少 + 给选择——① 已挂限价就等价格回参考价成交；② 想立即成交就用现价市价/限价（说明做空价/做多价变差多少、盈利空间变化）。
- 然后按**实际可能成交的价**重算浮盈、止损距离、赔率，不要拿过时的参考价算。

**信号时间用「写文件 + 响铃」的实测时刻（alert.sh 内部 date，写进 ring-log.csv），不用 AI 组织信号文本的估算时间**（2026-07-17 用户立，2026-07-18 随「写文件 + 响铃合并进 alert.sh、写文件先于响铃」新规修订）：用户能开始 App 操作的时刻是**响铃那一刻**，记录信号时间要用它才反映真实可执行时点。新规下 `alert.sh` 把「① 写信号文件 ② 响铃 + 写 ring-log」合进同一个脚本调用——写文件与响铃几乎同时刻，ring-log.csv 里 `alert.sh` 内部 `date` 记下的时间戳（精确到秒）即权威的「写文件 + 响铃时刻」。实操顺序：AI 组织信号内容（算好止损 / 仓位 / 参考价 / 价格范围）→ **调 `alert.sh`（stdin 传信号内容：写文件 + 响铃 + 写 ring-log）** → 在对话输出信号表格 → 响铃后立即 monitor 取实测价（按 ring-log.csv 时间戳匹配响铃时刻）→ 把实测价 + ✅/❌ append 到该条信号记录。AI 在信号标题里写的 HH:MM:SS 用调 `alert.sh` 前的最近时间（接近写文件时刻），以 ring-log.csv 时间戳为权威。原 2026-07-17 记的 10:20 / 10:23 是写信号估算时间（偏早），新规下写文件与响铃同时、时间戳不再偏早。

**开仓价给范围，响铃后超出范围则假设开仓失败**（2026-07-17 用户立）：
- 开仓 🟢 / 加仓 🔵 信号给**价格范围**（不是单一价），范围按盈亏比逻辑算（2026-07-17 用户精确规则，取代之前 ±容差估算法）：
  - **做多**：下限 = 止损价；上限 = (止盈价 + 止损价) ÷ 2（盈亏比=1 的入场价）。
  - **做空**：上限 = 止损价；下限 = (止盈价 + 止损价) ÷ 2。
  - 推导：盈亏比 = 1 ⇔ 入场价 = (止盈+止损)/2；做多入场越接近止损（下移）盈亏比越大、做空越接近止损（上移）盈亏比越大；范围保证盈亏比 ≥ 1 且入场在止损价正确侧（方向对）。⚠️ **区间端点顺序按方向**：做多 [止损, 盈亏比=1价]（止损在下）、做空 [盈亏比=1价, 止损]（止损在上）——数学区间 [a,b] 要求左端≤右端，不能固定写成 [盈亏比=1价, 止损]（那是做空顺序；做多时盈亏比=1价>止损，写成 [盈亏比=1价, 止损] 就 a>b、不成立）。例：做空 止损1310/止盈1200 → 盈亏比=1价=(1310+1200)/2=1255，范围 [1255, 1310]；做多 止损843/止盈880 → 盈亏比=1价=861.5，范围 [843, 861.5]（2026-07-18 用户纠正严谨化）。
- 响铃后按 ring-log.csv 时间戳（alert.sh 写文件 + 响铃时刻，精确到秒）+ **立即 monitor 取实测价**（响铃后马上取 ≈ 响铃时刻价，勿延迟几分钟才取）；**预估成交价 = 响铃时刻实测价**（不是信号参考价）——用户能开始 App 操作的时刻是响铃、响铃后市价就是用户能拿到的成交价（市价单即时撮合），故用响铃时刻实测价算浮盈 / 止损距离 / 盈亏。判：实测价在范围内 → 假设成交；超出 → 假设开仓失败、告知用户。

这条是 [[signal-is-real-trading]]（信号=真实交易）的执行细节：真实交易就得关心是否真成交、信号时间是能否操作的时刻，不能假设挂了就成、也不能用 AI 内部时间冒充用户可操作时间。

**signals 文档记录每个开仓/加仓的成交状态 + 预估实际成交价**（2026-07-17 用户立）：响铃验证后，在 signals 对应记录标注 **✅ 成交 / ❌ 未成交**（含响铃时刻实测价 + 范围判定）+ **预估实际成交价 = 响铃时刻实测价**（市价单即时报这价成交；算浮盈/盈亏用它，不用信号参考价）。失败的仓不计入持仓、不基于它算浮盈/止损/平仓盈亏。
