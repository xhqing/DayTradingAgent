**信号 = 真实交易，非演练。** 用户拿信号后会**真实下单**（真金白银），不是「演练 / 实战模拟」。2026-07-16 用户明确：「你发出的信号不是演练，我会真实交易」「请认真对待」。

**Why**：2026-07-15 skill 立的「自主演练模式」假设 AI 为积累实战样本可「敢于发不完美信号、不过度保守」；但用户实际是真实交易——2026-07-16 SOXL 做空（置信度定为 🟡 中、已标注超大单逆势买风险）仍按演练逻辑发了信号，用户实亏 ~$120（超 max_loss，3x 滑点超损）。用户据此澄清并收紧：信号是真金白银，必须严格把关质量，不能为「积累样本 / 抓演练机会」发置信度不足的信号。**那笔亏损的判断失误在 AI：在真实交易背景下，已标注明确风险（超大单逆势买）的中置信信号本就不该发。**

**How to apply**：
- **信号质量优先于信号数量**：不为「多积累一个样本」「抓演练机会」「想开仓」而降低门槛。开仓必须过两道闸（[[open-position-thresholds]]：仅 🟢 高置信 + 赔率 ≥1.5），不达标就不发、宁可观望一整场。
- 原演练框架的机制层保留（假设执行不查 positions、信号先对话后记录、复盘由用户触发、AI 不下单用户手动执行）；但「敢于发不完美信号」「为积累样本发」「赔率/入场点不完美也发」的倾向**作废**——换成真实交易的严谨。
- 每个开仓信号发出前自问：用户会真按这个下单、真亏真赚——风控（max_loss、止损位、3x 滑点预留、止损不被噪音扫）是否实算到位、置信度是否真到 🟢 高。
- 已同步 trade skill（2026-07-16 立、2026-07-18 术语清理）：节标题统一为「信号模式总则」（原「自主演练模式升级」节名废弃）、第3条「敢于发不完美信号」作废改「质量优先、不为积累样本发不完美信号」。关联 [[open-position-thresholds]]、[[stop-loss-trigger-no-signal]]。
