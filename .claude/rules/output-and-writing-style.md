# 输出与写作风格：对话高密度，文件重清晰

**语义清晰是底线**（对话和文件都适用），区别只在密度：对话追求高信息密度、文件追求语义明确。不能为精简牺牲语义。

## 对话输出：高信息密度

最大化单位时间 / 单位 token 的信息量——精简、不重复。

- 实时场景（如盯盘早盘）尤其要快、密、短：状态秒变时，思考长 + 输出重复 → 用户收到结论时情况已变、结论失配。
- 执行：一行带过（关键值 + 变化 + 判断），无变化不重复展开；不复述细节、不重复"无信号 / 无变化"理由。
- 信号类输出：先落记录后发对话，对话极简（emoji + 关键字段）。

## 写文件：语义清晰优先于密度

写入文件（skill / 文档 / 配置说明）必须语义明确、无逻辑矛盾，**可以降低信息密度、用更多文字把因果说清楚**——文件追求准确清晰，不追求精简。

- 写完读一遍，检查"逻辑跳跃 / 表象当核心 / 术语歧义"（例：把"没法止损锁利"写成"过夜"——过夜是表象，无人盯盘、没法及时止损锁利才是核心）。
- 常见陷阱：成本价 vs 成交价（做空成本为负数）、上方 / 下方方向歧义、为短而牺牲准确。
