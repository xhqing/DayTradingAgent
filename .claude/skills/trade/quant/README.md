# quant/ — 可回测的量化策略

本目录是 trade skill 下**专门放可回测的量化策略**的目录（2026-07-14 立）。以后所有可回测的量化策略都在这里，不散落到别处。

## 定位（与 trade skill 主流程的关系）

trade skill 的核心是 AI（LLM）做港股 / 美股日内交易——但 **LLM 本人不可回测**（拟人、不可复现、前视偏差与人同理）。本目录的策略是**独立于 LLM 的、确定性的代码策略**，可被历史数据回测，回测结果标定其信号可信度，盘中作为 AI 的**加权输入**之一。

一句话：**这里的东西不取代 AI 的判断，而是给 AI 提供一个"历史验证过的、可量化的投票员"。** 详见 `../SKILL.md`「表现验证：量化策略信号作为可回测的加权输入」。

## 核心设计原则

1. **策略只分类，置信度由回测赋予**：策略模块输出信号方向 + **类别签名**（不自己打分）；置信度 = 该类别在回测里的**历史胜率 / EV**（由回测脚本统计）。策略模块自评的"分"和回测脱节、不可信，故不用。
2. **双用途、同一套代码**：回测和盘中实时调**同一份**策略逻辑，保证回测出的可信度认得上实时信号。
3. **只用可回测因子**：日 K（OHLCV）+ 分钟 K（OHLCV）。长桥分钟 K 太浅，但**富途 OpenD 有约 5.5 年分钟 K（2021-01 起）**——分钟级日内策略可回测。资金流 / 盘口 / 换手率无历史，仍不进策略。数据源详见 `data-sources.md`。

## 目录结构（约定）

```
quant/
├── README.md              # 本文件（目录用途 + 规矩）
├── data.py                # 长桥日 K 分页拉取（喂策略 + 回测共用）
├── backtest.py            # 回测驱动器：拉数据→逐根喂策略→统计→出「类别→可信度」查表
└── strategies/            # 每个策略一个文件，纯逻辑（不含置信度计算）
    └── trend_breakout.py  # 例：5 日顺势 + 放量突破回踩不破
```

## 怎么加新策略

1. 在 `strategies/` 加一个 `.py`，实现统一接口：输入 K 线序列 → 输出 `{方向, 类别签名, ...}`（**不含置信度**，置信度由回测赋予）。
2. 在 `backtest.py` 注册该策略 → 跑回测 → 生成该策略各类别的**历史胜率 / 赔率 / EV / 样本数**查表。
3. 盘中实时调用：策略出信号 + 签名 → 查回测表得该类历史胜率（= 自评置信度）→ AI 综合进最终信号。
4. 配套：样本量太少的类别（如 <20 次）置信度打折；查无的新类别 → 弃权，不干扰 AI。

## 数据约束（实测，见 `../SKILL.md`）

- **长桥日 K**：单次上限 1000 根，分页拼接（每页≈4 年）可拉十几年；港股美股均支持。符号格式 `代码.市场` 后缀（`02800.HK` / `SPY.US`），用富途前缀格式 `HK.02800` 会报 `invalid symbol`。
- **富途 OpenD 分钟 K**：约 5.5 年（2021-01-21 起），1m/5m/15m 均有；单次 `max_count` 上限 10 万根，分页拉全量。**分钟级日内策略回测的主力源。** 符号 `市场.代码` 前缀（`HK.02800`）。详见 `data-sources.md`。
- **资金流 / 盘口 / 换手率**：无历史，不进策略。
