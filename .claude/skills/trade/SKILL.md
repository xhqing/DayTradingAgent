# Trade

AI 在长桥（主交易）+ 老虎/富途 OpenD（港股辅助数据源，Level2 盘口/经纪队列 + WebSocket 推送）上做港股/美股日内交易的执行规范。所有规则来自 2026-07-03 实战复盘，**交易亏钱可接受，但事实错误和违规操作是 bug、不可接受**。

## 触发条件

当用户提出以下任一意图时激活本 skill：
- 盯盘 / 看盘 / 分析标的走势 / 给买卖信号
- 发开仓信号 / 平仓信号（AI 不直接下单，只发信号由用户在长桥 App 执行）
- 交易复盘 / 算盈亏 / 算成本 / 算手续费
- 切换账户 / 查账户资金 / 查持仓 / 查额度
- 任何涉及长桥 CLI 或老虎 SDK 的实盘操作

## 第一原则：事实先验证，再说结论（最高优先级）

这是一切交易判断的地基，违反这条等于在沙地上盖楼。**凭推测当事实输出，比交易亏钱更不可接受**（2026-07-03 用户多次纠正）。具体执行：

1. **接触任何标的代码，第一件事 `longbridge static <sym> --format json` 查 `name` 字段确认真身**，再谈分析。港股代码本是 5 位数字（如 02800、00005、03688），无语义规律，凭记忆会认错（0823 是领展 REIT 不是汇丰、汇丰是 0005）。代码 + 中文名一起写进汇报。
2. **算术必须实算**，不许心算报数（如以为"400 不是 200 的倍数"，实际 400÷200=2）。
3. **交易日历用 `longbridge trading days <MARKET> --start <D> --end <D>` 查**，不许凭记忆说哪天开盘。港股 7/3+7/5 周末休市、美股 7/3-7/4 独立日休市等都曾搞错过。
4. **CLI / API 字段先跑 `--help` 或 `--schema` 看真实结构再用**，不许凭印象猜字段名（max-qty/margin-ratio 返回是 `[{field,value}]` 键值对数组不是对象，按对象字段取会得 None）。
5. **手续费/利率/汇率等数字必须有据**，禁止估算报数（详见下"成本核算"）。
6. **币种符号必须分明**（2026-07-04 用户规矩）：港币一律写 `HK$` 或 `HKD`，**禁止用裸 `$` 表港币**——`$` 在本 skill 默认指美元（账户净资产、买力等多以 USD 计价）。长桥 `assets` 返回的金额不带币种前缀，报数时须据字段（`currency`/`HK$` 可用港币 vs `USD available`）判定币种后正确标注，美元写 `$` 或 `USD`，港币写 `HK$`/`HKD`，混用会让人误把港币当美元（或反之）。

## ⚠️ 信号模式总则（2026-07-07 用户铁律，最高优先级，覆盖下方一切旧下单规则）

本 skill 已从「AI 自动下单」改为 **AI 只发信号、用户手动执行**。AI 不再直接碰任何订单。下方硬性护栏 / 自查清单里凡涉及「下单 / 挂单 / order buy/sell」的旧描述，均以此总则为准——AI 不下单。

1. **禁止自动下单**：AI 不得调用 `longbridge order buy/sell/cancel` 等任何下单 / 撤单命令。所有买卖由用户拿信号后在长桥 App 手动执行。
2. **禁止设置条件单**：AI 不得挂任何条件单 / 止损单 / 止盈单（MIT / MO / LIT / LO 一律不挂）。止损价只在开仓信号里**计算并告知**，由用户在 App 自行挂止损。
3. **只输出两类信号**：
   - **开仓信号 🟢（绿色）**：必含 ① 标的（代码 + 中文名，已 `static` 核验）② 方向（做多 / 做空）③ 具体开仓量（止损反推 + lot_size 取整）④ 开仓参考价（`quote` 实价）⑤ **计算出的止损价**（技术位）⑥ 止盈参考 ⑦ 单笔最大损失金额（≤ `config.max_loss`）⑧ 一句话策略依据 ⑨ **账户**（实盘：主账户 / 日内融，AI 据当日策略决定，见「实盘账户对比与选择」；模拟盘训练期写模拟盘）。
   - **平仓信号 🔴（红色）**：必含 ① 标的 ② 方向（平多 / 平空）③ 平仓参考价 ④ 一句话理由（动能减弱 / 回踩不破前低 / 资金流转正 / 触及止损 / 到硬平仓时点 等）。
4. **信号 emoji 标识**：开仓 🟢 / 平仓 🔴 / 移动止损 🟡。**信号直接对话表格输出（多 emoji 醒目），不写 MD**（见「信号格式」）。
5. **盯盘仍由 AI 做**（最高频采样，见「盯盘节奏」）→ 产生信号；**执行权全在用户**。
6. **AI 职责边界**：分析、盯盘、算仓位 / 止损价、发信号（**直接对话表格 emoji，不写 MD**）。信号发出后等用户反馈成交。**不记账、不实时算盈亏**（复盘时再算）。
7. **实盘账户由 AI 决定（2026-07-10 用户授权）**：用户让做**实盘交易**时，**用哪个实盘账户（主账户 / 日内融）由 AI 决定**——据当日策略（能否盯到收盘 / 是否看好持仓到收盘 / 标的保证金差异）推荐，写入信号「账户」字段，用户照着在该账户 App 执行。两实盘资金可随时互划；AI 仍不下单。决策细则见「实盘账户对比与选择」。模拟盘训练期仍用模拟盘。
8. **信号时效性：慢决策 / 查询项前置到盯盘启动，发信号零额外延迟（2026-07-13 用户铁律）**：盘中盘面秒变，发信号前任何临时查询 / 思考 / 确认都会让信号发出来时盘面已大变、结论失配。故以下事项**全部前置到盯盘启动时（preflight 阶段）做好，盘中触发信号那一刻直接拿来用、不再临时做**：
   - **汇率**：盯盘启动时就查好当日 USD/HKD 汇率（`exchange-rate`），盘中算美股仓位 / 止损额度直接用预备好的数，**不临时查、不临时换算**。
   - **账户选择**（主账户 / 日内融 / 模拟盘）：盯盘启动据当日策略定好（见「实盘账户对比与选择」），写入后续信号「账户」字段，盘中发信号时**不再思考用哪个账户**。
   - **账户买力**：用户对自身资金心里有数、或盯盘启动时已判好（`assets`/`max-qty`），盘中**不再反复判断账户买力够不够**。
   - **不反复确认**：拿到最新 `quote`/盘口数据后直接算 + 发信号，**只要信号够快盘面就不会大变**；不为求稳而反复 `quote` / 反复核实拖时间。相关核实（positions 无持仓、标的分类、quote 实价）已在 preflight / 启动序列做过，发信号时不重复（平仓前核实持仓仍保留，见「用户执行反馈」）。
   - 一句话：**信号触发那一刻零额外思考 / 零额外查询**——能前置的全前置，盘中只做「拿最新数据 → 算 → 发」。

## 平仓行为与术语定义（2026-07-07 定义，2026-07-10 加移动止盈）

**两类平仓行为**（开仓都是主动；平仓分主动 / 被动）：

| 行为 | 触发方 | 盈利 | 亏损 |
|---|---|---|---|
| 主动开仓 | AI 开仓信号 → 用户 App 操作 | — | — |
| 主动平仓 | AI 平仓信号 → 用户 App 操作 | **主动锁利** | **主动锁亏** |
| 被动平仓 | 止损/止盈条件单触发（用户在 App 挂） | **被动锁利** | **被动锁亏** |

**关键规则**：

1. **条件单主要用止损；移动止盈是动态锁利方法**（2026-07-10 用户加）：AI 发**移动止盈建议**（止盈位可**上移**锁更高目标 / 也可**下移**到更近位快速锁利、提胜率），用户在 App 操作（到价平仓或挂动态止盈单）。⚠️ **不看好且主动平仓来不及时**（AI 思考→发信号→用户 App 执行有延迟，主动平仓信号可能错过最佳点）→ **移动止盈下移到当前价附近**，价格稍动即被动触发锁利，比主动平仓信号更快兜底。固定止盈条件单仍不发。
2. **AI 给止损价 + 移动止盈价**：随持仓阶段给移动止损价（防守、锁下限）+ 移动止盈价（进攻、锁上限），用户 App 挂/调。AI 不碰任何条件单。
3. **平仓信号 = 主动平仓**：AI 发平仓信号 = 主动离场（情况不对 / 到目标），盈利=主动锁利、亏损=主动锁亏。
4. **条件单触发 = 被动平仓**：价格到止损/止盈位自动触发。**被动锁利两法**：① 移动止损朝盈利方向移（锁下限）；② 移动止盈（朝盈利更深处移锁更高目标 / 朝成本回移快速锁利，双向）。

**AI 的输出四种**（2026-07-10 加移动止盈）：① 开仓信号 🟢 ② 平仓信号 🔴 ③ 移动止损建议 🟡 ④ 移动止盈建议 🟠。**不挂任何条件单（用户 App 挂）**。

## 硬性护栏（发信号前逐条自检，违反不得发信号）

1. **单笔最大损失限额，开仓前先定止损位再反推仓位**（2026-07-04 用户新规，取代旧的"10% 净资产"仓位护栏）。**限额数值读 `config.json` 的 `risk.max_loss_per_trade`（默认 1000 HKD），不写死在文档**：
   - **先定止损位**：止损位 = 技术位（前低 / 支撑位 / 跌破的关键价位），AI 据盘面定（`config` 的 `stop_loss_method=technical`）。固定百分比模式见配置。
   - **算每股最大损失**：做多 = 开仓价 − 止损价；做空 = 止损价 − 开仓价。止损价 ≥ 开仓价（做多）属方向错，不开仓。
   - **反推仓位**：原始仓位 = 最大损失额度 ÷ 每股最大损失。港股按 `lot_size` 向下取整到整手倍数（不足 1 手不开仓）；美股可零股直接取整。
   - **美股换算**：HKD 额度 ÷ 当日真实汇率（`exchange-rate` 命令查，勿估）= USD 额度，再 ÷ 每股最大损失。
   - **周期**：`loss_limit_scope` 默认 `per_trade`（每笔独立，无日累计上限，但 AI 自行在连亏后控制开仓频率）。
   - **仓位无净资产比例上限**：止损位近则仓位大、远则仓位小，均允许，只要单笔最大损失 ≤ 配置额度。仓位资金来源：先用账户现金买力（`max-qty` 查），**现金买力不足时可启用融资补足差额**（融资规则见下「融资使用规则」），但融资部分需纳入净盈亏成本（利息见「成本核算口径」第 2 条）且受融资授信额度与风控（margin_call/risk_level）约束，超授信或触发风控则按可用额度下调或放弃。
2. **止损价必须在开仓信号里给出，由用户在 App 挂止损**（2026-07-07 信号模式）：AI 不挂任何单。开仓信号里的止损价 = 技术位（前低 / 支撑 / 跌破关键价），必须明确算出写进信号，并在信号末尾提示「开仓后请在 App 挂止损 @ $Y」。历史教训（2026-07-06 模拟盘）：CLI 条件单 MIT / MO / LIT 全 Rejected、状态 `VarietiesNotReported` 不可信，依赖 AI 挂单 = 止损失效裸奔——信号模式后此风险交由用户 App 止损兜底（App 止损单已验证有效）。止损位一旦定了就是纪律，不因盘中情绪滑动。
3. **AI 不下单，订单类型由用户在 App 选**（2026-07-07 信号模式）：信号里给「参考价」（`quote` 实价），用户据其在 App 用限价 / 市价执行。AI 不再规定 LO / MO，也不再挂任何条件单。
4. **信号前 `quote` 确认实价**作信号参考价；成交状态由用户执行后反馈，AI 据反馈用 `positions` 核实。
5. **不记账**（2026-07-09 用户要求）：信号只写 `signals/YYYY-MM-DD.md`，不算实时盈亏、不写 `ai_trade_ledger.csv`；复盘时用户要求再统一算当天盈亏。
6. **一次最多持仓一个标的（硬规矩）**：发开仓信号前必须 `positions` 检查用户当前无持仓；有持仓只在现有标的上发加 / 减 / 平信号，绝不发新标的。**平仓信号发出、用户平仓后，提示用户在 App 撤销该标的所有未触发止损 / 止盈 / 挂单**（2026-07-06 教训：残留条件单下次开盘误触发）。
7. **标的范围**：仅个股（含 ADR）+ ETF。**术语习惯（2026-07-06 用户）**：用户说「港股」「美股」时**默认含 ETF**（港股市场交易的 ETF + 个股都算），除非单独强调「个股」。**禁止衍生品**（期权/窝轮/CBBC/期货），即使账户开通了期权行情也不碰。港股偏好 ETF（免印花税），美股个股 ETF 均可。**杠杆/反向 ETF（含 2x/3x）允许**（2026-07-06 确认 2x、2026-07-09 扩展含 3x：形态是 ETF，内部虽用衍生品实现杠杆，仍视为 ETF 允许交易，如两倍做多海力士/三星电子 ETF、SOXL 三倍做多半导体 ETF；⚠️ 3x 波动更大、止损要更紧；下单前核实代码 + 流动性）。

## 融资使用规则（2026-07-04 用户新规）

建仓时若仓位所需资金**超过账户当前现金买力**，允许启用融资（margin）补足差额，不再"超买力即放弃"。规则：

1. **先用现金买力，再补融资**：`max-qty` 查现金买力对应可买量；仓位需求 ≤ 现金买力 → 不动用融资；仓位需求 > 现金买力 → 现金部分正常下单、超额部分走融资，下单时账户须已开通融资授信（`assets` 的 `max_finance_amount` > 0 且有 `remaining` 额度）。
2. **融资额度是硬上限**：融资部分不得超过 `remaining` 可用融资额度。下单前用 `assets`（或 `margin-ratio`）核实剩余授信，超授信则按可用额度下调仓位或放弃，不许透支。
3. **风控指标不得越线**：下单后须复核 `init_margin`/`margin_call`/`risk_level`。`margin_call > 0` 或 `risk_level` 非 Safe 即视为已触发预警，**不得再开新仓或加仓**，先减仓降风险。
4. **融资利息计入净盈亏**：融资部分按账户整体融资余额日结利息（对账单 `interests` section 查，见「成本核算口径」第 2 条），按持仓周期分摊到该笔交易。日内融账户收盘前自平可避免被动强平利息与隔夜利息。
5. **日内融账户天然带杠杆**：日内融账户（account_no 见 `accounts.json`）的买力本身就是融资属性（买力数值已含杠杆，见 `accounts.json`），使用时等同已启用融资，同样受上述额度与风控约束，且必须遵守收盘前 `flat_before_close_mins` 自平（见「实盘账户对比与选择」）。
6. **止损位仍是第一约束**：融资放大了亏损的绝对额，但**单笔最大损失 ≤ `risk.max_loss_per_trade` 的护栏不变**——止损位反推出的仓位若需融资才买得起，才用融资；若止损位反推的仓位本身就小到现金买力够用，则不应主动加杠杆。

|---|---|---|

- 港股：收盘 16:00 HKT → 最迟 **15:45 HKT** 强平
- 美股：收盘 16:00 ET → 强平 = **美东 15:45 = 北京夏令时 03:45 / 冬令时 04:45**

**账户选择决策（AI 据当日策略）**：
1. **美股晚上、看好走势、想睡不盯 → 用日内融**（巧妙用法）：设好止损条件单、**不主动平、直接去睡**。两种结果都不劣于主动止损：① 走势坏 → 触发止损单（按止损价平，亏损受控）；② 走势没坏 → 收盘被券商强平，**强平价优于止损价**（没触发止损 = 价格仍在止损有利侧，强平更划算）。主账户**不会被券商强平**——用户睡觉时仓位无人盯，**没法及时止损或锁利**（价格大跌没人平，这才是核心问题；"过夜"只是表象）。故美股早盘用主账户 → AI 必须在北京 24:00（≈美东 12:00）前发平仓信号、用户主动平仓再睡（2026-07-11 用户）；想睡到收盘就用日内融（强平兜底）。
2. **能盯到收盘的日内交易 → 两账户均可**：主账户主动平、日内融可主动平或被强平。选保证金要求更低（买力更大）的那个。
3. **不看好 / 要严控离场 → 主动平**（两账户均可，不等强平）。
4. **只做日内、从不过夜（2026-07-10 用户铁律）**：所有市场、所有账户的开仓**当日必须平掉**，从不开仓过夜。主账户虽不强平，美股持仓也必须 AI 主动发平仓信号当日平（不能借"不强平"过夜）；日内融则收盘前被强平。"设止损睡"属美股当日（收盘前强平了结），不算过夜。

> 日内融强平是**软约束但当硬规矩执行**：依赖强平有不可控成交价 + 费用风险，故"看好设止损睡"仅在**确信走势、止损位合理**时用；否则仍应主动平。

## 交易策略纪律（2026-07-03 实战教训）

**核心思想：数学期望 EV ≥ 0**（2026-07-06 用户立，最高优先级）：一切交易决策围绕 **EV ≥ 0**（至少不亏），EV > 0 且越大才挣得多。EV 由多笔交易的**胜率 × 赔率**决定——这是能否盈利的地基，方向/风控/纪律都服务于它。

- **开仓基准**：预估这笔交易的胜率，只有 **胜率 > 50% 且赔率 > 2** 才考虑开仓（EV = 胜率×赔率 − 败率×1，此时 EV > 0；胜率或赔率任一不达标则不开仓）。
- **置信度分级——EV 之上的第二道闸（2026-07-13 借鉴 OhMyTradingTeam 置信度框架）**：EV 判据是"这笔值不值得做"的数学门槛；置信度是"我对这个判断有多大把握"的可靠度分级。**EV 达标后，再看置信度决定仓位轻重或是否发信号**：
  - **🟢 高置信**：信号各维度一致（趋势 + 量价 + 资金流 + 宏观基调同向）、入场点建立在已确认事实（企稳 / 真站稳）、无临近重大事件 → 正常仓位。
  - **🟡 中置信**：部分维度矛盾或待确认（趋势顺势但量价未配合、资金流与盘口背离、临近数据日但方向不明）→ **降仓位**（如减半）或等确认。
  - **🔴 低置信**：关键维度矛盾（量价背离 + 资金流反向）、横盘无明确方向、临近重大事件（CPI / 非农 / FOMC / 财报）方向难判、依据是赌突破 / 赌反转（预测非已确认）→ **不发信号**，等局面明朗。
  - 置信度写入开仓信号「🚦 置信度」字段。⚠️ 置信度是**软闸**（中置信可发但降仓位），不取代硬护栏和 EV 判据（那两道是硬性不发）；置信度低到 🔴 才不发。
- **赔率公式**：初始预估赔率 =（止盈线 − 成本线）价差 ÷（成本线 − 开仓止损线）价差。开仓前必须先定止盈线 + 止损线，才能算出赔率、再判开仓。
- **移动止损不撤旧单（2026-07-06 用户）**：移动止损时**只挂新止损单（更紧、更有利位），不撤旧止损单**——旧单留在更宽价位做备用兜底。原理：移动止损单总朝"减小亏损/增加利润"方向调（新单位于更有利价位），价格必先触发新单；若新单失效（Rejected/未承接），旧单兜底。撤旧单 = 自废一层保护。
  - ⚠️ **移动止损只能单向移动**（2026-07-10 用户强调）：只朝**有利方向**（亏损减小 / 利润增加），**永不回撤**——做多止损只上移、做空只下移（做多下移止损 = 扩大风险，禁止）。**移动止损既锁损（亏损时限制亏损）又锁利（盈利时上移到盈利区）**，覆盖持仓全周期。这是它与**移动止盈**的根本区别：**移动止损 = 止损价单向调整 + 锁损&锁利；移动止盈 = 止盈价双向调整 + 只锁利**。⚠️ **止损价、止盈价从开仓就同时设定、时时刻刻都在**（开仓信号含止损价 + 止盈价）；**止盈价始终在盈利方向**（做多高于成交价 / 做空低于成交价）所以只锁利；**止损价初始在亏损方向**（做多低于成交价 / 做空高于成交价，锁损）**→ 可单向移到盈利方向锁利**。（注：用"成交价"而非"成本价"作参照——空头成本价 = 成交价的相反数即负数，标的价始终正数，正负比"高于/低于"无意义。）
- **移动止损浮盈大时锁利、不只保本（2026-07-08 SOXL 教训）**：浮盈 ≥2/股 时，移动止损应放到**盈利区**（锁利 1+/股），不只放到成本位（保本）。2026-07-08 做空 SOXL：成本 162.84、浮盈 $96（现价 160.37、2.47/股）时只把止损移到成本 162.84（保本），结果 SOXL 反弹到 162.84 保本出、**$96 浮盈全回吐**。正确做法：浮盈 2.47/股 时止损应移到 **161 附近**（锁利 1+/股 ≈ $39+），反弹触发锁利、不回吐浮盈。原则：**浮盈越大、止损越要往盈利区挪**（保本只是浮盈小时的底线，浮盈大了必须锁利）。
- **锁利大小要动态权衡、不机械固定（2026-07-10 用户纠正）**：浮盈中移动止损，**锁利多（止损紧）= 防回吐但被小反弹扫、错过续顺势的大赔率**（实际剩余赔率降）；**锁利少（止损松）= 让利润奔跑但回吐风险**。不机械"锁 2+/股"或任何固定值，而是**实时预估剩余赔率（止盈空间 ÷ 止损距离）+ 续顺势 vs 反弹概率，动态选锁利位使 EV 最大**。判据：标的波动大（3x ETF）止损要偏松（防小波动扫损）；大趋势强、续顺势概率高 → 锁松让利润奔跑；超跌、反弹概率高 → 锁紧防回吐。**看好后市 → 微锁利 + 宽止盈**（止损放成交价附近、做多略高 / 做空略低于成交价；**止盈位放远、给足空间让利润奔跑**）：被扫也微利保本不亏、不扫吃大涨；微锁利=止损近 + 宽止盈=止盈远 = **高赔率**（止盈空间大 ÷ 止损距离小），配合看好的高胜率使 EV 最大化。⚠️ **微锁利必须搭配宽止盈**，否则"防住回撤却早早平掉大涨潜力"自相矛盾。微锁利程度取决于预判（越看好越松至成本持平、甚至略下方给回撤空间）。⚠️ **使用边界**（2026-07-11 用户）：**「微锁利 + 宽止盈 + 睡觉让奔跑」仅限美股早盘（北京晚上）+ 日内融 / 模拟账户**（用户睡觉时段、靠日内融收盘强平兜底）；**港股盘中（北京白天，用户与 AI 同步盯盘）不用此法**——止盈止损**时时刻刻据盘面主动动态调整**，不睡觉。**锁利位是浮盈中持续重估的变量，不是一次定死**。
- **主动平仓不等固定止盈（2026-07-06 用户教训，最高优先级）**：浮盈中若**情况不对**（动能减弱/资金流边际转正/价量背离/震荡不前/滞涨），**随时 LO 主动平仓锁利**，**绝不死等预设止盈位（如 2 倍赔率）**。用户原话："主动平仓不一定要等到两倍赔率，情况不对随时平仓。" 移动止损只是被动兜底，主动判断离场才是锁利主手段——执着等固定止盈会把已到手的利润还给市场。第一单做空 MU 浮盈 +$212、近5分钟资金流一度转正 +987（动能减弱信号）时未主动平仓，后反弹变亏损 −$92，典型错过。
- **回踩不破前低 = 最佳平仓时刻（2026-07-06 用户）**：做空时价格**回踩前低不破**（= 下跌动能衰竭、买盘承接站住）→ 这才是技术面上最佳主动平仓时刻，优先于固定止盈位；做多时回踩前高/支撑不破 = 加仓或持有时刻。错过这个点位往往由盈转亏。
- **移动止损总原则**：持仓全程移动止损线；当**止损价移过成交价到盈利方向**（做多止损价 > 成交价 / 做空止损价 < 成交价）时，这笔的胜负已锁定（最差不亏），之后让利润奔跑、止盈价朝盈利更深处调。
- **资金流入但价格不涨 = 卖盘更强（2026-07-07 港股复盘，最高优先级）**：资金流（capital inflow）只有伴随**价格同步突破**才是多头确认。若资金持续流入但价格多次冲阻力不过（价量背离），说明上方卖盘在出货、把买单全吃了——这是**离场 / 观望信号，不是入场信号**。2026-07-07 美团第二笔误把「资金流入」当成「突破在即」，七次冲 78.50 失败，亏 −HK$125。
- **阻力三次不过不赌突破（2026-07-07 港股复盘）**：一个阻力被测试 ≥3 次都不破 = 强阻力确认，下次突破概率随测试次数**下降**（不是上升，每次失败都在验证卖盘强度）。想做多就等「真放量突破站稳 + 回踩阻力转支撑不破」，不要在阻力下方提前埋伏赌突破。
- **做多合格入场只有两种（2026-07-07 固化）**：① 回踩支撑企稳反弹（如 2026-07-07 美团 77.50 支撑企稳，+HK$990）；② 突破后回踩阻力转支撑不破。「赌突破」不在其中——前两者依据**已发生的事实**，后者依据**预测**。
- **量价正向确认作为开仓前置（2026-07-13 借鉴 Livermore 量价法则）**：上面「资金流入但价格不涨 = 卖盘更强」讲的是**背离**（资金进但价不涨 = 卖盘在出货，离场信号）；这里补**正向配合**（开仓确认信号）：顺势**做多要求放量**（上涨带量 = 买盘真主动）、**做空要求放量下跌或缩量反弹**（下跌带量 = 卖盘真主动）。**无量突破不可信**——价涨但量没跟上多半是诱多，不追。开仓前看一眼近几分钟量价是否配合，不配合就等。
- **不在关键点之外追单（2026-07-13 借鉴 Livermore 关键点法则）**：与「做多合格入场只有两种」呼应——那条讲**在哪入场**（回踩支撑 / 突破回踩），这条讲**为何不追远离关键位的拉伸**：价格已远离关键支撑 / 阻力位（涨了一大段没回调 / 跌了一大段没反弹）时**不追**，因为此时 EV 变差——① 离止损位远 → 仓位被迫缩小或止损被迫放松；② 拉伸越大 → 均值回归概率越高 → 胜率下降。等价格回到关键位附近（回踩 / 反抽）再进。一句话：**只在关键位附近交易，远离关键位不追**。
- **区间震荡市要切换区间交易、不执念破位做空（2026-07-08 SOXL 复盘教训，最高优先级）**：当价格在区间反复（如 SOXL 162-170 **多次假破位 / 假突破**），说明是**区间震荡**而非趋势——多次假破位本身就是「支撑有效」的信号。此时强支撑（多次测试不破）= **做多信号**（向阻力看，赔率好），强阻力 = **做空信号**（向支撑看），**而不是等破位顺势**。2026-07-08 SOXL 162 三次假破位都反弹，162 是强支撑、应在 162 做多（止损 160、止盈 170、赔率 4；实际反弹到 168 +3.7%），但我执念「板块弱做空」、反复等破 162 做空（三次假破位全错过 + 一笔保本）。教训：**≥2 次假破位/假突破 = 区间信号，立即切换区间交易（支撑买 / 阻力卖），不执念单一方向**；"二次确认避免假破位扫损"是对的，但更要从假破位读出"支撑强→可做多"。
- **事后结果不反推决策对错（2026-07-07）**：评判一笔交易看「当时依据是否站得住」，不看事后盈亏。2026-07-07 第二笔事后看美团摸到 78.50，但中途 77.45 接近 77.90 止损、大概率被扫，依据（赌突破）本就不成立 → 决策错，与事后无关。
- **信号模式入场更要「已确认」而非「预测」（2026-07-07）**：用户 App 执行有延迟，赌突破 / 赌反转的标的若回落，延迟放大损失。入场点应建立在已发生的事实（企稳 / 真站稳）上，给执行延迟留容错。
- **杠杆 ETF 反弹强且久，斐波那契 0.5 回撤不是顶（2026-07-10 07709 复盘）**：2x 杠杆 ETF 超跌反弹力度大、持续久，判「反弹遇阻」不能在 0.5 回撤位（50%）就下结论，要等 0.618（61.8%）甚至 0.786 / 前高附近**多次测试不过**才算遇阻（与「阻力三次不过」同理）。2026-07-10 做空 07709（两倍做多海力士）：跌势 98.86→90.68（区间 8.18），0.5 位=94.77、0.618 位=95.74；反弹到 94.6（0.5 位）我就喊遇阻，实际穿透 0.5、0.618 一路到 **97.50（83% 回撤）**。过早判遇阻 → 踏空或被扫。**回撤位算法**：位 = 低点 +（高点−低点）×比例（0.5 / 0.618 / 0.786）。
- **止损位与入场点是 EV 综合权衡，不是「止损越远越安全」（2026-07-10 07709 复盘）**：做空时止损近=赔率高但胜率低（易被反弹扫），止损远=胜率高但赔率低。必须按 EV=胜率×赔率−败率×1 综合判，不能只看赔率或只看「不被扫」。2026-07-10 07709 在 94 做空：止损 96 赔率 2.5 但胜率≈0（午后 97.5 必扫，EV=−1）；止损 99 胜率 85% 但赔率仅 1.0（EV=+0.7 薄）——差入场点上怎么选止损都别扭。**正解是等更好的入场点**：午后 97.5 遇阻做空、止损 99、赔率 5.7、胜率 80%（EV=+4.36），「止损近且不被扫」与「赔率高」同时成立。核心：**找高胜率 + 高赔率同时成立的入场点，而非在差位置纠结止损远近**。
- **领先指标持续走强时不做空主标的（2026-07-13 SOXL 实盘复盘，亏 $81.7 / HK$637）**：做空 ETF 时，其重仓成分 / 领先指标（如 SOXL 之于 MU/SNDK）是方向驱动。**领先指标持续创新高时，ETF 阻力即使"三次不过"确认也会被突破**——领先指标强势是破阻力的动能，机械套用「阻力三次不过」会吃亏。2026-07-13 实盘做空 SOXL 40 股 @ 171.82、止损 173.5：SOXL 173 阻力三次不过、MU 冲 932.99 回落 929（看似失败新高）时入场，但 MU 随后继续创新高 936.47，带 SOXL **持仓仅 50 秒**即冲破 173.5 止损，亏 $81.7。教训：① **领先指标持续创新高时不做空主标的**，等领先指标明确见顶（连续 ≥2-3 轮不创新高 + 买卖比持续负）再空——这条优先级高于 ETF 自身的阻力确认；② **「失败新高」判断要严**：单次冲高回落 ≠ 见顶，需连续多轮确认（与「杠杆 ETF 反弹强且久」同理，反弹 / 强势持续力易被低估）；③ **3x ETF 急涨时市价止损有滑点**（本单 173.5→173.8625，+0.36 / +0.21%），止损位预留滑点 buffer、或 max_loss 预算计入滑点（本单超 $80 预算即因滑点）；④ **纪律一致性**：盯盘中前几轮因"MU 强不空"反复撤信号，这一轮 MU 仍强（932）却破例空——临场不为"想开仓"而放宽自己立的标准。
  - **【2026-07-13 第二单升级，累计两单亏 HK$899】**同日第二单做空 SOXL 30 股 @ 173.41、止损 174.5：**已遵守①**（等 MU 连续 4 轮不破 940.45 才空），但 MU **再次创新高 943.50**、SOXL 33 秒涨破止损，亏 $33.6。教训升级：① **「连续 N 轮不创新高」也不是见顶充分条件**——强反弹市领先指标震荡整理后会再突破，"不创新高 N 轮"只说明短期整理、不说明趋势反转；见顶须确认**明确转跌**（连续阴线 + 跌破前回调低点），非仅"不创新高"；② **强反弹市（暴跌后 V 型反弹）不做空反弹**——反弹力度反复超预期（SOXL 165.6→177.66 +7.3%），做空反弹策略本身在高波动反弹日失效、反复被扫；③ **做空等下跌趋势确认**（标的破前低 / 关键支撑、反弹明确结束），不抢反弹顶部（抢顶 = 赌反转、胜率低）；④ **上涨途中做空执行价变差**：信号发出 SOXL 171.57 → 用户执行 173.41（3 分钟续涨 1.84），反弹中做空越拖越不利（与「信号时效性」总则呼应，但此处延迟来自执行端 + 价格本身续涨，非 AI 反复 quote）。

1. **方向大于风控**：逆势交易风控再细也难赚。建仓前必看近 5 日 K 线趋势，只在上升趋势回踩或下降趋势反抽时顺势入场，不接飞刀。
2. **资金流 > 盘口 > 分时**：盘口墙可造假（挂大单压价/闪挂闪撤），但持续的大单净流入要真金白银、难造假。`longbridge capital` 看资金流。盘口偏空但大单持续净流入 → 倾向"洗盘"持有；盘口强但大单净流转负 → 警惕"对倒出货"。判别假/真盘口墙：连续 2-3 轮采样（每轮 30-50 秒）观察是否"持久挂单 + 价格真涨" vs "闪挂闪撤 + 价格不动"。
3. **加仓要等回踩不破支撑**，不追在突破点。强势突破不给回踩则放弃加仓，宁错过不追高。
4. **没有明确顺势信号时宁可空仓**，不为扳回亏损强行交易（"最大化收益率 ≠ 最大化交易次数"）。
5. **连亏不是停手理由，信号不合格才是**：AI 无情绪干扰，两连亏后不带"扳回"执念、仍按"顺势 + 资金流 + 主动止盈"纪律执行，有正期望（2026-07-03 前两单逆势亏 -301，后两单顺势 +66 扳回 22%）。
6. **找标的必须先过热度榜（2026-07-07 用户铁律，覆盖旧"优先"措辞）**：凡是用户让"找 / 选 / 筛交易标的"或问"有什么可做的 / 看看机会"，**第一步必须跑富途 `get_hot_list(market)` 热度榜**筛选，**不许凭下方关注列表或 AI 记忆直接定标的**。流程：热度榜头部 → 叠成交额 / `volume_ratio` 筛流动性 → 叠顺势过滤（近 5 日趋势）→ `static` + `classify_hk_security.py` 核实代码与类型 → 才进入分析（方法细节见下方子条「热度判断方法」）。原 2026-07-06「优先近期热点」升级为硬性必须。**备选关注列表（仅作热度榜之外的参考，不作直接选标的依据；会变，下单前必须核实）**：半导体存储链——海力士 / 三星电子杠杆 ETF、美光科技 MU。⚠️ **杠杆/反向 ETF（2x）合规待用户确认**：形态是 ETF、但内部用衍生品实现杠杆，与护栏⑦「禁衍生品」有张力，交易前需用户明确是否纳入允许范围 → **已确认允许（2026-07-06，2026-07-09 扩展含 3x 如 SOXL）**，下单前核实代码 + 流动性即可。
   - **热度判断方法（2026-07-06 实测）**：富途 `get_hot_list(market)`（热度榜，关注度+资金维度）比 `get_top_movers_rank`（纯涨幅榜）更靠谱；选热度标的必须叠 `volume_ratio`/成交额筛流动性，排除「振幅高但盘口薄」的小盘题材（实测港股涨幅榜 02526 振幅 283% 那种 = 陷阱区）。热度榜头部 + 流动性好 + 顺势 = 高效机会。实测美股热度榜头部为半导体存储链 MU/SNDK/NVDA/SOXL。

## 盯盘节奏

**盯盘持续性（2026-07-13 用户铁律）**：用户让盯盘后，**未叫停就持续盯，不中途停下等用户**。靠 cron 循环（`/loop`，session-only）+ 密采样维持，任务不因单轮对话结束而终止——无信号时一行带过继续盯，不暂停问"要不要继续"。方向研判、找标的、盯信号、发信号、平仓全程连续推进，直到用户喊停或收盘。

### 盯盘启动方向研判（盯盘第一步定当日基调，2026-07-13 借鉴 OhMyTradingTeam 多视角框架日内化）

盘中找标的（hot_list）是"看今天什么在动"，但缺一步先定"今天大方向偏多还是偏空"。本研判用**多维度独立打分后综合**，定当日方向倾向，缩小找标的范围（只做顺势方向）——避免见涨追涨、逆势开仓。

**时机**：盯盘启动的**第一步**，在 `preflight` 之后、`hot_list` 之前——**无论用户何时叫醒盯盘（盘前预热或已盘中）都先跑一次**，产出：一句话"今日港股/美股偏多 / 偏空 / 中性震荡"。盘前启动用隔夜外围 + 指数结构判基调；盘中启动数据更全（有实时资金流 / 量价），照样走这五维度。核心是"**先定方向再找标的**"，时点不限、不是只在盘前做。

**五个维度独立给倾向**（每维偏多 +1 / 中性 0 / 偏空 −1，最后求和定向）：

| 维度 | 看什么 | 来源 |
|---|---|---|
| **宏观基调** | **最近一根**美股日 K 收盘涨跌 / 美 10 年期国债收益率方向 / 美元指数 / VIX；今日有无央议息、CPI/非农、重大政策 | 长桥 `quote` 查美股主指数 / ETF；外围消息看新闻。⚠️ **"最近一根"≠"昨夜"**：周一开盘时最近一根是**上周五**（周末美股休市），节假日后开盘同理跨假期——须跨周末/假期核对期间事件（地缘/政策/数据），不许说"昨夜"、不许忽略中间跨度（2026-07-13 用户纠正） |
| **技术趋势** | 目标市场指数近 5 日 K 线结构（上升 / 下降 / 横盘）、距关键支撑阻力远近 | `scripts/kline.py <指数代码> 5` |
| **量价** | 指数近几日涨跌是否带量（量价配合 = 趋势健康，背离 = 虚弱） | kline 量柱 |
| **资金流** | 目标板块 / 标的资金净流向（盘前数据有限，开盘后用 `capital` 复核） | 长桥 `capital`（盘中）|
| **事件催化** | 当日有无个股 / 板块级事件（财报、产品发布、政策落地）——事件日波动大、方向不定 | 财经日历 |

**综合定向**：
- 和 ≥ +2 → 今日偏多，找标的优先找顺势做多机会（hot_list 后叠"近 5 日上升趋势"过滤）。
- 和 ≤ −2 → 今日偏空，优先找顺势做空机会。
- 其余 → 中性震荡，按区间交易思路（支撑买 / 阻力卖，见「交易策略纪律」区间震荡条），降低开仓预期。

**重要边界**（避免误用）：
1. **本研判是"大方向参考 + 选标的过滤"，不是盘中信号**。方向偏多不代表立即追多，仍要等盘中合格入场点（回踩企稳 / 真突破站稳，见「做多合格入场只有两种」）。
2. **宏观基调维度较慢**，只影响"今日整体偏多偏空"的背景判断，不进实时盯盘链路（盯盘看秒级盘口，宏观是背景）。
3. **事件催化日**（CPI / 非农 / FOMC / 财报）方向不定但波动剧：优先上 WebSocket 密采样（见下「重要数据日优先 WebSocket」），开盘后等波动消化再入场、不抢第一波。
4. 中性震荡日**不是不开仓**，而是切区间交易（支撑买 / 阻力卖）、降单笔预期、止损更紧。

**盯盘标准启动序列**：`preflight` → **方向研判（见上「盯盘启动方向研判」）** → `hot_list` → `static`+`classify` 核实标的 → `snapshot`+`kline` 分析 → `monitor` 密采样盯。脚本有问题再临时改，勿从零重写（2026-07-10 用户要求前置固化，起因盯盘准备磨蹭半小时）。⚠️ **启动时一并把当日汇率 / 账户选择 / 买力备好**（见「⚠️ 信号模式总则」第 8 条）——这些是盘中会用到但耗时的慢决策 / 查询项，全部前置到启动阶段，盘中触发信号时直接用、零额外延迟。

- **采样频率按场景分级（2026-07-06 铁律 + 2026-07-09 修订）**：股市价格瞬息万变——**有持仓 / 早盘 / 盘面异动时采用最高采样频率**（实测 10 秒间隔、每条命令 6-8 轮，6×10=60 秒 < harness 超时 120 秒，无 API 限流），**此时禁止降频**（曾设每 10 分钟 cron 盯盘 = 止损失效未及时发现的根因之一）。开盘首小时（港股 09:30-10:30 / 美股 21:30-22:30）急涨急跌尤其要密。**仅当无持仓且盘面平稳时**，可按 `config.json` 的 `monitoring.calm_interval_min`（默认 10 分钟，配合 /loop）间歇巡检；一旦开仓 / 进入早盘 / 出现异动，立即切回最高频率。约束：`rounds_per_call × sample_interval_sec` 必须 < `harness_timeout_sec`，否则分多条命令执行。
- **早盘密采样（实测工程参数，2026-07-07 补）**：开盘首小时（港股 09:30-10:30 / 美股 21:30-22:30）流动性集中、急涨急跌最快，10-30 秒内可能完成一次冲高又回落的完整动作——稀疏采样只看到结果，分不清「真突破」与「冲高诱多」，早盘胜负往往在这几秒到几十秒内决定。工程实测结论（2026-07-03 盘后）：
  - **采样间隔统一取 10 秒**（用户 2026-07-07 取 config / 正文 / memory 方案三处口径最小值；早期方案 A `sleep30×3`≈62 秒、方案 B `sleep20×5`≈84 秒均工程可行，已被更密的 10 秒取代）。配置读 `config.json` 的 `monitoring.early_session.sample_interval_sec`。
  - **API 限流不是瓶颈**：长桥 OpenAPI 连续 30/60 次调 quote/depth/capital（无 sleep）实测约 1.6 次/秒、**零失败零限流**；10 秒间隔负载（0.1 次/秒）远低于压力测试速率，间隔小不会触发限流。
  - **物理下限约 15 秒/决策回合**：纯 CLI 调用约 0.6 秒/次，带「采样→读→决策」的回合约 15 秒；要秒级响应只能上 WebSocket 推送（见下条）。
  - **采样输出只留关键字段**（price / bid1 / ask1 / 3 档差 / 大单净流），**不打印完整 JSON**：单轮 quote+depth+capital 约 1KB，连续几十轮会撑爆上下文，须定期摘要而非全量输出。
  - **高信息密度输出（2026-07-09 用户要求，最高优先级）**：思考 + 输出**精简、不重复**，最大化单位时间 / 单位 token 信息量。原因：早盘价格秒变，AI 思考长 + 输出重复 → 用户收到分析 / 信号时盘面已变、结论失配。执行：① 盯盘报告一行带过（价 + 变化 + 判断），无变化不重复展开；② 信号直接对话表格 emoji（不写 MD）；③ 不复述盘面细节、不重复"无信号"理由；④ **早盘（开盘首小时）尤其要快、密、短**。⚠️ **语义明确、无逻辑矛盾是底线（文件 + 对话都适用，2026-07-11 用户）**。区别只是密度：**对话输出**追求高密度短，但**前提是语义明确**（不能为精简牺牲语义）；**写入文件（skill / memory）**优先语义清晰、可降低密度用更多文字把逻辑说清楚。
  - **美股开盘前 15 分钟启动盯盘**（2026-07-08 回退）：不做盘前交易（护栏 10），但开盘前 15 分钟（夏令时 21:15 / 冬令时 22:15）启动盯盘预热，等 21:30 开盘。
  - **重要数据日优先 WebSocket**：非农 / CPI / FOMC 等日子开盘波动极剧，10 秒间隔仍可能漏秒级信号，这类日子直接上老虎 WebSocket（方案 C）。
  - **开盘前 5 分钟验证 token**：港股 09:25、美股 21:25 跑 `/tmp/lb.sh longbridge check`，避免开盘才发现 token 失效。
- **优先用老虎 WebSocket 推送替代轮询**（港股）：7/6 开盘后用 PushClient 订阅 quote/tick/depth，毫秒级实时，突破长桥 sleep 轮询瓶颈。代码骨架见 `tiger-websocket.md`。

## 成本核算口径（净盈亏必须全成本）

**净盈亏 = 毛盈亏 − 手续费 − 融资利息分摊**，三项分别列示不混算。

1. **手续费禁止估算**：`order detail`/`executions` **无费用字段**，真实手续费只能从对账单 `stock_trades` section 查，且**日对账单 T+1 生成**。当天交易只报毛盈亏，净盈亏标"待次日对账单"。用 `longbridge statement list --type daily --start-date <昨日>` 拿 file_key，再 `statement export --file-key <KEY> --section stock_trades`。
2. **融资利息**：从对账单 `interests` section 查，按账户整体融资余额日结，无法精确归到单笔，按持仓周期分摊（仍以对账单日利息为据，不凭空估）。日内融收盘前自平可避免被动强平利息。
3. 报盈亏时未查到费用的部分明确标"待查"，绝不编数字填进去。

## 用户执行反馈（信号模式协作协议）

AI 不下单，所有订单 / 成交均为用户在 App 操作。协作流程：

- **AI 发信号 → 用户 App 执行 → 用户反馈成交（或 AI 主动 `positions` 查询）→ AI 转入盯持仓**（不记账，复盘时再算）。
- **持仓方向与数量以用户 App 实际成交为准**：AI 发平仓信号前必须 `positions` 查用户实际持仓，确认有持仓后发平仓信号（平仓即全部平掉，不指定数量；不凭开仓信号里的量假设）。
- **用户未按信号执行属正常**（用户有最终决定权）：AI 发现 `positions` 与自己发出的信号不符，不告警、不催促，按实际持仓继续盯盘发信号。
- **用户自行买卖（非按 AI 信号）**：AI 通过 `positions` / `order list` 发现持仓变化后纳入盯盘，按新持仓发持有 / 平仓信号。
