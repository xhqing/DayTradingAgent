# Trade

AI 用富途 OpenD / 老虎 SDK（盯盘行情数据主力，Level2 盘口 / 经纪队列 + 资金流 + WebSocket 毫秒级推送）做港股 / 美股日内交易的执行规范。**信号模式：AI 只发买卖信号、用户手动执行，AI 不管账户、不下单、不核实资金 / 持仓**。盯盘数据走富途 + 老虎（资金流富途更优：含 super 超大单 + 分钟序列 + 十大买卖经纪）。所有规则来自 2026-07-03 实战复盘，**交易亏钱可接受，但事实错误和违规操作是 bug、不可接受**。

## 触发条件

当用户提出以下任一意图时激活本 skill：
- 盯盘 / 看盘 / 分析标的走势 / 给买卖信号
- 发开仓信号 / 平仓信号（AI 不直接下单，只发信号由用户手动执行）
- 交易复盘 / 统计信号胜率 / 赔率 / EV
- 任何涉及富途 OpenD / 老虎 SDK 的盯盘数据操作

## 第一原则：事实先验证，再说结论（最高优先级）

这是一切交易判断的地基，违反这条等于在沙地上盖楼。**凭推测当事实输出，比交易亏钱更不可接受**（2026-07-03 用户多次纠正）。具体执行：

1. **接触任何标的代码，第一件事用富途 `get_market_snapshot([code])` 查 `name` 字段确认真身**，再谈分析。港股代码本是 5 位数字（如 02800、00005、03688），无语义规律，凭记忆会认错（0823 是领展 REIT 不是汇丰、汇丰是 0005）。代码 + 中文名一起写进汇报。
2. **算术必须实算**，不许心算报数（如以为"400 不是 200 的倍数"，实际 400÷200=2）。
3. **交易日历用富途 `request_trading_days` 查**，不许凭记忆说哪天开盘。港股 7/3+7/5 周末休市、美股 7/3-7/4 独立日休市等都曾搞错过。
4. **API 字段先实跑一次看真实返回结构再用**，不许凭印象猜字段名（如富途 `get_market_snapshot` 返回 DataFrame、老虎 `get_stock_briefs` 也是 DataFrame，按对象属性直取会得 None）。
5. **手续费 / 利率 / 汇率等数字必须有据**，禁止估算报数，查不到标"待查"绝不编数字填进去。
6. **币种符号必须分明**（2026-07-04 用户规矩）：港币一律写 `HK$` 或 `HKD`，**禁止用裸 `$` 表港币**——`$` 在本 skill 默认指美元。报数时须据标的所在市场正确标注币种，美元写 `$` 或 `USD`、港币写 `HK$`/`HKD`，混用会让人误把港币当美元（或反之）。

### 🔴 信号模式总则（2026-07-15 初立自主盯盘循环 → 2026-07-16 修订：演练改真实交易；最高优先级，覆盖本段第 4-8 条及下方「用户执行反馈」「信号输出与持仓检查」相关旧描述）

⚠️ **2026-07-16 用户关键澄清：信号 = 真实交易（用户拿信号真实下单、真金白银），不是演练。** 故原「自主演练循环 / 敢于发不完美信号抓演练机会」框架中「为积累样本降低门槛」的倾向作废，改为真实交易的严谨——**质量优先于数量**（见第 3 条）。机制层保留（假设执行不查 positions、信号写文件先于响铃、复盘由用户触发、AI 不下单用户手动执行）。原 2026-07-15 框架 5 条中，第 3 条 2026-07-16 已重写、其余机制保留：

1. **假设执行、不查 positions**：AI 发出任何信号（开仓/加仓/减仓/平仓/移动止损）后，**直接假设用户已按信号执行**（即假设已持仓/已平仓），**不查 positions、不等用户反馈**。用户是否真执行是用户自己的事（用户可能不下单，但 AI 当成已成交继续盯）。用户若真实操作致持仓变化，会**临时主动告知**——只有这时 AI 据实调整；用户没说就按假设持仓盯。
2. **自主循环、不打断用户**：AI 自主跑「盯盘→发信号→记录信号→总结经验教训」循环，不中途停下问用户、不等确认（用户会睡觉/离开）。**港股盘中 + 美股盘中（美东 09:30 开盘后到 16:00 收盘）自主盯**；美股盯到**用户发出停止盯盘指令**或**美东 16:00 收盘**（取先到）为止，停盯前必须主动平仓、不留单（2026-07-19 修订，撤销原「美东 12:00 固定停盯」+「留单 vs 主动平二选一」，见护栏 10）。盯盘由用户指令启动、AI 不自主启动。
3. **质量优先、不为积累样本发不完美信号（2026-07-16 修订，原「敢于发不完美信号」作废）**：信号是真实交易（用户真实下单、真金白银），**质量优先于数量——不为「积累样本 / 想开仓」降低门槛**。开仓必须过两道闸（仅 🟢 高置信 + 赔率 ≥ 1.5，见「交易策略纪律」开仓基准与置信度分级），不达标就不发、宁可观望整场。原 2026-07-15「演练阶段敢于发信号、赔率/入场点不完美也发」作废（2026-07-16 SOXL 中置信信号致用户实亏 ~$120 后用户立）。
4. **信号记录到 signals/ 目录（用户下单基准，时序：先写文件、再响铃、最后对话输出；港美分开）**：信号文件是用户下单的基准（不是对话表格——对话会被推走、易丢失），故**决定发信号后第一个动作是写信号文件**——通过 `alert.sh` 一次性完成「① 写入对应市场文件 + ② 响铃并记录响铃时刻」（脚本调用方式与完整时序见「信号格式 · 🔔 信号时序总则」），**之后再在对话输出信号表格**。港股 `signals/YYYY-MM-DD-HKT-signals.md`、美股 `signals/YYYY-MM-DD-ET-signals.md`（2026-07-15 用户立：港美信号分开记、文件名带 HKT/ET 时区标识，便于分别复盘）。记录必含：发信号时间（**精确到秒**）、标的、方向、量、参考价、止损、止盈、最大损失、置信度、依据、风险提示（不写账户 / 资金）。`signals/` 目录 2026-07-15 恢复（2026-07-10 曾 archive 删除）。
5. **复盘由用户指令触发、盘中不复盘（2026-07-16 立，覆盖原「自主复盘」）**：AI **不自主复盘**——**盘中不发任何复盘内容、盘后也不主动复盘**，只有**收到用户明确的复盘指令**才做复盘——按「复盘分析」小节逐项统计（**样本总量 / 胜率 / 赔率 / EV% / 平均每单盈利金额** + 用户临时指定的其它分析），并点评各笔依据是否站得住、入场止损止盈是否合理、错过哪些机会。原 2026-07-15「盘中及收盘后 AI 自主复盘、不等用户要求」作废。⚠️ 注意区分：**记录信号**（发完信号 append 到 `signals/` 文件）是发信号后的即时动作、**不是复盘**，仍照常做；被限的是「事后分析评价」性质的复盘。
6. **AI 职责边界（2026-07-15 用户细化，最高优先级）**：AI **只负责 ① 发信号 ② 记录 ③ 事后复盘统计（胜率 / 赔率 / EV）**，**不管账户、不下单、不核实资金 / 持仓**。仓位按 `risk.max_loss_per_trade` 反推作风控建议（与账户无关）。复盘仍是 AI 的核心职责之一，但**由用户指令触发、盘中不做**（2026-07-16 修订，见第 5 条）——收到复盘指令后，统计每个信号从发出到平仓的**胜率、赔率、EV**（EV=胜率×赔率−败率×1，核心判据；**不是简单收益率**）。**默认假设：所有标的都可做空、资金总是够用**——发做多 / 做空信号均不犹豫可行性，专注方向与入场点判断。

6. **AI 职责边界**：分析、盯盘、算仓位 / 止损价、发信号（**发信号第一个动作 = 调 `alert.sh` 写入对应市场文件 + 响铃，之后再在对话表格 emoji 输出**：港股 `signals/YYYY-MM-DD-HKT-signals.md` / 美股 `signals/YYYY-MM-DD-ET-signals.md`；完整时序见「信号格式 · 🔔 信号时序总则」）。假设执行、不等用户反馈成交（详见「信号模式总则」第 1、6 条）。**不记账、不实时算盈亏**（复盘时再算）。

8. **信号时效性：慢决策 / 查询项前置到盯盘启动，发信号零额外延迟（2026-07-13 用户铁律）**：盘中盘面秒变，发信号前任何临时查询 / 思考 / 确认都会让信号发出来时盘面已大变、结论失配。故以下事项**全部前置到盯盘启动时（preflight 阶段）做好，盘中触发信号那一刻直接拿来用、不再临时做**：
   - **汇率**：盯盘启动时就查好当日 USD/HKD 汇率（富途 `get_market_snapshot` 或汇率接口），盘中算美股仓位 / 止损额度直接用预备好的数，**不临时查、不临时换算**。
   - **不反复确认**：拿到最新 `quote`/盘口数据后直接算 + 发信号，**只要信号够快盘面就不会大变**；不为求稳而反复 `quote` / 反复核实拖时间。相关核实（标的分类、quote 实价）已在 preflight / 启动序列做过，发信号时不重复。假设执行、不等用户反馈成交（见「信号模式总则」第 1 条）。
   - 一句话：**信号触发那一刻零额外思考 / 零额外查询**——能前置的全前置，盘中只做「拿最新数据 → 算 → 发」。

## 信号格式（2026-07-18 更新：信号时序改为「先写文件、再响铃、最后对话输出」）

### 🔔 信号时序总则（2026-07-18 用户立，覆盖原「先对话、后记录」，最高优先级）

用户以 `signals/` 下信号文件的内容为**下单基准**——信号最终落地、用户实际参照下单的是信号文件，不是对话里的表格（对话流会把信号推走、易丢失，文件才是稳定可回查的基准）。故**决定发信号后，第一个动作是写信号文件**，确保基准先建立：

1. **第一个动作：调用 `alert.sh`**（写文件 + 响铃合并进同一脚本，脚本内两步顺序固定）：
   - **① 把信号内容写入对应市场文件**（港股 `signals/YYYY-MM-DD-HKT-signals.md` / 美股 `signals/YYYY-MM-DD-ET-signals.md`）；
   - **② 响一声对应类型的提示音 + 记录响铃时刻到 `signals/ring-log.csv`**（时间精确到秒，事后按时间戳匹配响铃时刻价）。
2. **第二个动作：在对话输出信号表格**（emoji + 表格呈现，供正在看对话窗口的用户即时看到）。

**为什么写文件先于响铃**：响铃是通知用户「文件已就绪、可去看文件下单」——文件必须先写好，用户被铃叫醒 / 吸引过去时，文件已是完整、可直接下单的内容；先响铃、文件没写好 = 用户听到铃却看不到内容。写文件在前、响铃在后，确保铃响那一刻文件已就绪。

**`alert.sh` 调用方式**（信号内容是多行 markdown，经 stdin 传入，避免命令行参数转义问题）：

```bash
cat <<'SIGNAL' | bash .claude/skills/trade/scripts/alert.sh <type> <market> [symbol] [note]
## 🟢🟢🟢 开仓 · <标的代码> <中文名> · 做多/做空 🟢🟢🟢
（完整信号内容：标题 + 表格，对话里呈现什么就往这里写什么）
SIGNAL
```

- `type`：`open`=开仓🟢/加仓🔵、`close`=平仓🔴/减仓🟠、`ts`=移动止损🟡（决定音色）；
- `market`：`HKT`=港股、`ET`=美股（决定写到哪个信号文件，**必填**）；
- `symbol` / `note`：可选，写入 `ring-log.csv` 备查（如 `US.MU`、`入场854/止损843`）。

⚠️ **脚本写文件前先校验**：信号内容为空（忘了 pipe）、缺 `type` / `market`、`market` 非 HKT/ET 时直接报错退出、**不写任何文件**——发信号时看 `alert.sh` 的退出码与 stderr 报错，确认基准文件真的写进去了、再继续后续动作。

**响铃后立即取实测价并补记（2026-07-17 立，时序随 2026-07-18 新规）**：响铃是用户开始 App 操作的时刻，响铃时刻的市价 ≈ 用户实际成交价。响铃后立即 `monitor` 取响铃时刻实测价（不依赖取价动作的及时性——可事后按 `ring-log.csv` 时间戳匹配富途历史 K 线 / quote 回查），把实测价以一行注释 append 到该条信号末尾（形如 `> 响铃时刻实测价：$P（HH:MM:SS）`），并据此判成交 / 失败（**2026-07-21 用户立新规、同日二次修订为不对称范围**）：成交判据 = 实测价是否落在**不对称成交范围**内（做多 [参考价 − 0.8R, 盈亏比=1 价] / 做空 [盈亏比=1 价, 参考价 + 0.8R]，R 与盈亏比=1 价定义见下条）：
- **成交价格范围（不对称、分方向，2026-07-21 用户二次修订）**：**做多 = [参考价 − 0.8R, 盈亏比=1 价]、做空 = [盈亏比=1 价, 参考价 + 0.8R]**。两条边界各是：
  - **R = 一个风险单位 = |参考价 − 止损价|**（做多 = 参考价 − 止损价、做空 = 止损价 − 参考价），与旧规相同、不变。
  - **盈亏比=1 价 = 实际盈亏比降到 1 的成交价临界 =（止盈价 + 止损价）/ 2**（使「到止盈的距离 = 到止损的距离」的中间价）。它落在参考价**朝盈利方向一侧**（做多在上方、做空在下方），离参考价多远取决于原赔率——原赔率越高、盈亏比=1 价离参考价越远（赔率=2 时恰在参考价 ± 0.5R 处、赔率=3 时在参考价 ± R 处）。
- **为什么不对称（成交价偏离对做多 / 做空的有利方向相反，故两端边界不同）**：**做多**——下沿参考价 − 0.8R 允许成交价低于参考价（买入更便宜 = 对做多有利、止损距随之变大），但不能低于此（成交价再低就离止损价只剩不足 0.2R、太容易被盘中波动扫损）；上沿盈亏比=1 价禁止成交价更高（成交价越高买入越贵、实际赔率降到 1 以下、不值得开）。**做空**对称反向——上沿参考价 + 0.8R 允许成交价高于参考价（卖出更贵 = 对做空有利），但不能更高（否则离止损价只剩不足 0.2R）；下沿盈亏比=1 价禁止成交价更低（卖出越便宜实际赔率降到 1 以下）。一句话：**成交价朝有利方向可偏离参考价最多 0.8R（给滑点容错）、朝不利方向只能偏到盈亏比=1 价（守住赔率 ≥ 1 的底线）**。
- **成交判据**：实测价落在对应方向的成交范围内 → 假设成交；**实测价超出此范围 → 开仓失败**（朝有利方向偏离超 0.8R = 离止损价太近易被扫；朝不利方向越过盈亏比=1 价 = 赔率 < 1 不值得开）。
- **示例（2026-07-21 中芯实例）**：做多参考价 76.6、止损 76.1、止盈 78.5 → R=0.5、盈亏比=1 价=(78.5+76.1)/2=77.3，成交范围=[参考价 − 0.8R, 盈亏比=1 价]=[76.6−0.4, 77.3]=[76.2, 77.3]；响铃实测 76.1 < 76.2 → 朝有利方向偏离 = 1R > 0.8R 上限（成交价已跌到止损价）→ 开仓失败。
- **区间符号约定**（2026-07-21 用户立）：`( )` 开区间不含端点、`[ ]` 闭区间含端点；`(a,b]` 左开右闭（左不含、右含）、`[a,b)` 左闭右开（左含、右不含）、`(a,b)` 双开、`[a,b]` 双闭。
- **开仓失败的后果**：开仓失败 = **未实际开仓 = 未设置止损单 = 仍属空仓状态**；信号文件标注"开仓失败、无开仓"、不记盈亏、不计为复盘样本 T、无需任何 App 操作（没挂止损单也就不用撤销）。
- **旧规作废**：① 2026-07-21 早先的「参考价 ± 0.5R（做多做空对称）」被本次二次修订取代——旧规对称、只卡「偏离参考价半个风险单位」，没区分偏离方向有利还是不利（有利方向被 0.5R 卡死、容错不够；不利方向却仍放到 0.5R、可能赔率已跌破 1）；新规按方向不对称（有利方向放到 0.8R 给足滑点容错、不利方向用盈亏比=1 价卡住赔率底线），更贴合「成交价偏离对多空盈亏影响相反」的实际。② 更早的「做多 [止损, 盈亏比=1 价] / 做空 [盈亏比=1 价, 止损]」亦作废（其有利方向那一端用到止损价 = 偏离整整 1R，放得太宽、成交价贴着止损价也算成交）。

## 平仓行为与术语定义（2026-07-07 定义，2026-07-18 信号体系调整：加仓/减仓替换移动止盈）

**两类平仓行为**（开仓都是主动；平仓分主动 / 被动）：

| 行为 | 触发方 | 盈利 | 亏损 |
|---|---|---|---|
| 主动开仓/加仓 | AI 开仓/加仓信号 → 用户 App 操作 | — | — |
| 主动减仓/平仓 | AI 减仓/平仓信号 → 用户 App 操作 | **主动锁利** | **主动锁亏** |
| 被动平仓 | 止损条件单触发（用户在 App 挂） | **被动锁利** | **被动锁亏** |

**关键规则**：

1. **止盈靠两类信号**（2026-07-18 取消移动止盈后）：① **主动平仓 🔴**（AI 判断到目标 / 情况不对，发平仓信号清仓）② **移动止损锁利 🟡**（浮盈中上移止损到盈利区，被动触发锁利）。不再有移动止盈信号。
2. **AI 只给止损价**：随持仓阶段给移动止损价（防守锁下限 + 浮盈时锁利），用户 App 挂/调。AI 不碰任何条件单。
3. **减仓 vs 平仓**：减仓 🟠 = 部分平仓（还有剩余，剩余仓位仍按原止损管理）；平仓 🔴 = 清空所有持仓。
4. **条件单触发 = 被动平仓**：价格到止损位自动触发。被动锁利靠移动止损朝盈利方向移（单向，锁下限）。

**AI 的输出五种**（2026-07-18 调整）：① 开仓 🟢 ② 加仓 🔵 ③ 减仓 🟠 ④ 平仓 🔴 ⑤ 移动止损 🟡。**不挂任何条件单（用户 App 挂）**。

**⚠️ 主动平仓 vs 被动平仓的信号边界（2026-07-16 立）**：
- **主动平仓 → 发 🔴 平仓信号**（AI 判断动能减弱 / 情况不对 / 到硬平仓时点等，需提示用户现在手动平仓）：第一个动作调 `alert.sh close`（写信号文件 + 响铃），之后再在对话表格输出（时序见「信号格式 · 🔔 信号时序总则」）。
- **被动平仓（止损 / 止盈条件单触发）→ 不发 🔴、不响铃**，只 ① 对话简短标记一句（「XX 止损/止盈 $Y 触发、成交 ~$Z、亏/盈 $M」）② signals 记录写明「触发价 / 盈亏 / 被动平仓」。原因：止损/止盈是用户在 App 挂的条件单，触发时 App 自动执行 + 通知用户，AI 发信号发出来时已触发、没有「提示用户去手动执行」的意义。
- 判别：AI 发出的是「让用户动手平」= 主动 🔴；「用户 App 条件单自己触发了」= 被动 → 只标记。

## 硬性护栏（发信号前逐条自检，违反不得发信号）

1. **单笔最大损失限额，开仓前先定止损位再反推仓位**（2026-07-04 用户新规，取代旧的百分比仓位护栏）。**限额数值读 `config.json` 的 `risk.max_loss_per_trade`（config 权威，当前固定 10000 HKD；preflight 启动时输出实际值，盘中算仓位以此为准）**：
   - **先定止损位**：止损位 = 技术位（前低 / 支撑位 / 跌破的关键价位），AI 据盘面定（`config` 的 `stop_loss_method=technical`）。固定百分比模式见配置。
   - **算每股最大损失**：做多 = 开仓价 − 止损价；做空 = 止损价 − 开仓价。止损价 ≥ 开仓价（做多）属方向错，不开仓。
   - **反推仓位**：原始仓位 = 最大损失额度 ÷ 每股最大损失。港股按 `lot_size` 向下取整到整手倍数（不足 1 手不开仓）；美股可零股直接取整。
   - **仓位留 buffer 覆盖成交价偏离（2026-07-17 用户立）**：信号参考价（发信号时）≠ 实际成交价（用户响铃后下单时刻），做空若成交价低于参考价、止损距 = 止损−成交价 会变大，按参考价把 max_loss 打满的仓位实际损失会超 max_loss。故开仓算仓位时**不打满 max_loss**，预留 buffer 吸收偏离，两选一（按标的波动选）：
     - **方案 A（简化，日常推荐）**：仓位 = max_loss ×（1−buffer）÷（止损−参考价），buffer 一般 **20-30%**（高波动标的放大）；即 max_loss 只用到 70-80%、留 20-30% 给成交价偏离。偏离落在 buffer 内 → 不用后处理；超出才减仓补救。
     - **方案 B（最严谨、保守）**：按价格范围下沿（最差成交价）算——做空范围 = [（止盈+止损）/2, 止损]，按下沿算止损距 =（止损−止盈）/2，仓位 = max_loss ÷（（止损−止盈）/2）；保证整个范围内任意成交价损失 ≤ max_loss，代价是仓位约为按参考价算的一半。
     - **两层兜底（能不动就不动）**：① 开仓留 buffer（方案 A/B）；② 响铃后按实际成交价重算损失——≤ max_loss（buffer 够覆盖偏离）→ 不动、正常持有；仅当偏离超 buffer、按实际成交价算仍 > max_loss 时才减仓补救（按实际成交价 + max_loss 重算合规仓位、减到不超）。
   - **美股换算**：HKD 额度 ÷ 当日真实汇率（`exchange-rate` 命令查，勿估）= USD 额度，再 ÷ 每股最大损失。
   - **周期**：`loss_limit_scope` 默认 `per_trade`（每笔独立，无日累计上限，但 AI 自行在连亏后控制开仓频率）。
   - **利润 = 赔率 R × 单笔最大损失，与标的振幅、价格贵不贵无关（2026-07-15 复盘沉淀，纠正错误认知）**：在「固定 max_loss 反推仓位」的框架下，盈利 = R × max_loss（R = 止盈距离 ÷ 止损距离），亏损 = max_loss。标的的价格只决定「买几股」（仓位股数 = max_loss ÷ 每股止损距离），**不决定赚多少钱**——便宜标的多买几股、贵标的少买几股，只要 R 相同，利润一样。故**选标的不能用「振幅大→利润大」「价格低→仓位好算→利润大」作理由**，这两条都是错的。
   - **做高赔率 R 有两条路，各有约束（2026-07-15 复盘沉淀）**：① 放大止盈——受日内可达范围（振幅）约束，止盈位得真到得了；② 缩小止损——受「不被盘中波动扫掉」约束，太窄会被随机噪音扫成白送。两条路都能抬 R，别只盯着「放大止盈受振幅限制」而忘了「缩小止损」这条路。
   - **杠杆同时影响止盈空间和止损空间——R 是增是减要看两者实际放大倍数、必须实算（2026-07-15 复盘纠正，二次修订严谨化）**：R = 止盈空间 ÷ 止损空间。杠杆（3x）会把止盈空间和止损空间**同时**推大，但 R 是增是减取决于两者放大倍数孰大——分子（止盈）放大倍数 > 分母（止损）则 R 升、相等则不变、< 则降，**不能笼统说「杠杆让 R 降低」或「振幅大让 R 升高」**（同倍数放大时 R 不变）。两者各自的实际值受不同约束：止损空间有「不被盘中波动扫掉」的下限（杠杆标的中盘中噪音大、下限更高，如 SOXL 需 3-5%，见下第 7 条 7/13 教训），止盈空间受「日内可达范围」约束。**故选标的不能凭振幅或杠杆想当然，必须针对每个标的、结合实际入场位 / 止损位 / 止盈位把 R 实算出来再比较**（示例估算、非定律：按各自合理止损宽度，MU 的 R 上限约 2.3-4、SOXL 约 1.6-2.7，换止损/止盈参数结论会变）。选标的的标准是「止损能设多窄（不被扫）× 止盈能到多远」，但要落到实算 R，而非振幅绝对值或杠杆倍数。
   - **仓位无比例上限**：止损位近则仓位大、远则仓位小，均允许，只要单笔最大损失 ≤ 配置额度。

## 交易策略纪律（2026-07-03 实战教训）

**核心思想：数学期望 EV ≥ 0**（2026-07-06 用户立，最高优先级）：一切交易决策围绕 **EV ≥ 0**（至少不亏），EV > 0 且越大才挣得多。EV 由多笔交易的**胜率 × 赔率**决定——这是能否盈利的地基，方向/风控/纪律都服务于它。

- **开仓基准（2026-07-16 用户修订：预估赔率门槛降为至少 1.5 倍）**：开仓必须同时满足 **胜率 > 50% 且 赔率 ≥ 1.5**（止盈距离 ≥ 1.5 × 止损距离）。EV = 胜率×赔率 − 败率×1；赔率 1.5 + 胜率 50% → EV = 0.5×1.5 − 0.5×1 = +0.25（正期望但较薄，更依赖胜率把关，故置信度门槛同步收紧到只发高置信，见下条）。⚠️ 赔率从原 ≥2（2026-07-15）降到 ≥1.5（2026-07-16）：**用置信度把关质量、放宽赔率抓更多机会**（置信度取代赔率做主把关）。⚠️ **赔率 ≥ 1.5 与「止损给足波动空间」不矛盾**：止损仍按技术位 + 标的波动合理设置（3x 杠杆 ETF 给 3-5%、见下方 2026-07-13 SOXL 止损太近被扫的教训，不被盘中噪音扫掉），赔率达标靠的是 **止盈空间 ≥ 1.5 × 止损空间** 这个比例，而不是强行收窄止损——做法是把止盈位放到止损距离的 1.5 倍以上、并确认它在当日日内可达范围内（受振幅约束，到不了的止盈位是空想）。移动止损仍用于浮盈后锁利与进一步放大利润，但**开仓那一刻赔率本身就要 ≥ 1.5，不能指望靠移动止损把一个低赔率的仓补成正期望**。
- **置信度分级——EV 之上的第二道闸（2026-07-13 借鉴 OhMyTradingTeam，2026-07-16 收紧：仅高置信才发开仓）**：EV 判据是"这笔值不值得做"的数学门槛；置信度是"我对这个判断有多大把握"的可靠度分级。**2026-07-16 修订（原「中置信降仓位可发」→「仅高置信发」）：开仓信号只在 🟢 高置信时发，🟡 中置信 / 🔴 低置信都不发开仓信号**（信号是真实交易，置信度把质量关、取代原「降仓位发」）。判级标准：
  - **🟢 高置信（唯一可发开仓的级别）**：信号各维度一致（趋势 + 量价 + 资金流 + 宏观基调同向）、入场点建立在已确认事实（企稳 / 真站稳）、无临近重大事件 → 发开仓信号。
  - **🟡 中置信（不发开仓）**：部分维度矛盾或待确认（趋势顺势但量价未配合、资金流与盘口背离、临近数据日但方向不明、已标注反向风险）→ **不发开仓信号**，等确认升到高置信再发（2026-07-16 SOXL 做空定中置信、已标注超大单逆势买风险仍发、致用户实亏 ~$120，用户据此收紧）。
  - **🔴 低置信（不发开仓）**：关键维度矛盾（量价背离 + 资金流反向）、横盘无明确方向、临近重大事件（CPI / 非农 / FOMC / 财报）方向难判、依据是赌突破 / 赌反转（预测非已确认）→ **不发信号**，等局面明朗。
  - 置信度写入开仓信号「置信度」字段（圆形彩色 emoji 🟢/🟡/🔴 标记，是表格内唯一可保留的 emoji，见「信号格式」）。⚠️ 2026-07-16 起**置信度是硬闸**（仅 🟢 发，🟡/🔴 都不发开仓），不再是「中置信可降仓位发」的软闸。置信度不取代硬护栏（标的分类、时段、max_loss 等仍硬性），但开仓门槛收紧到「🟢 高置信 + 赔率 ≥1.5」两道闸全过才发。
- **赔率公式**：初始预估赔率 =（止盈线 − 成本线）价差 ÷（成本线 − 开仓止损线）价差。开仓前必须先定止盈线 + 止损线，才能算出赔率、再判开仓。
- **移动止损不撤旧单（2026-07-06 用户）**：移动止损时**只挂新止损单（更紧、更有利位），不撤旧止损单**——旧单留在更宽价位做备用兜底。原理：移动止损单总朝"减小亏损/增加利润"方向调（新单位于更有利价位），价格必先触发新单；若新单失效（Rejected/未承接），旧单兜底。撤旧单 = 自废一层保护。
  - ⚠️ **移动止损只能单向移动**（2026-07-10 用户强调）：只朝**有利方向**（亏损减小 / 利润增加），**永不回撤**——做多止损只上移、做空只下移（做多下移止损 = 扩大风险，禁止）。**移动止损既锁损（亏损时限制亏损）又锁利（盈利时上移到盈利区）**，覆盖持仓全周期。2026-07-18 取消移动止盈后：止盈只靠 ① 主动平仓 🔴（到止盈目标主动清仓）+ ② 移动止损锁利 🟡（浮盈时上移止损到盈利区被动触发）——不再有独立的止盈价条件单。⚠️ **止损价从开仓就设定、时时刻刻都在**（开仓信号含止损价，初始在亏损方向锁损 → 可单向移到盈利方向锁利）。（注：用"成交价"而非"成本价"作参照——空头成本价 = 成交价的相反数即负数，标的价始终正数，正负比"高于/低于"无意义。）
- **移动止损浮盈大时锁利、不只保本（2026-07-08 SOXL 教训）**：浮盈 ≥2/股 时，移动止损应放到**盈利区**（锁利 1+/股），不只放到成本位（保本）。2026-07-08 做空 SOXL：成本 162.84、浮盈 $96（现价 160.37、2.47/股）时只把止损移到成本 162.84（保本），结果 SOXL 反弹到 162.84 保本出、**$96 浮盈全回吐**。正确做法：浮盈 2.47/股 时止损应移到 **161 附近**（锁利 1+/股 ≈ $39+），反弹触发锁利、不回吐浮盈。原则：**浮盈越大、止损越要往盈利区挪**（保本只是浮盈小时的底线，浮盈大了必须锁利）。
- **锁利大小要动态权衡、不机械固定（2026-07-10 用户纠正）**：浮盈中移动止损，**锁利多（止损紧）= 防回吐但被小反弹扫、错过续顺势的大赔率**（实际剩余赔率降）；**锁利少（止损松）= 让利润奔跑但回吐风险**。不机械"锁 2+/股"或任何固定值，而是**实时预估剩余赔率（止盈空间 ÷ 止损距离）+ 续顺势 vs 反弹概率，动态选锁利位使 EV 最大**。判据：标的波动大（3x ETF）止损要偏松（防小波动扫损）；大趋势强、续顺势概率高 → 锁松让利润奔跑；超跌、反弹概率高 → 锁紧防回吐。**看好后市 → 微锁利 + 宽止盈**（止损放成交价附近、做多略高 / 做空略低于成交价；**止盈位放远、给足空间让利润奔跑**）：被扫也微利保本不亏、不扫吃大涨；微锁利=止损近 + 宽止盈=止盈远 = **高赔率**（止盈空间大 ÷ 止损距离小），配合看好的高胜率使 EV 最大化。⚠️ **微锁利必须搭配宽止盈**，否则"防住回撤却早早平掉大涨潜力"自相矛盾。微锁利程度取决于预判（越看好越松至成本持平、甚至略下方给回撤空间）。⚠️ **使用边界**（2026-07-11 用户，2026-07-16 补强平终点）：**「微锁利 + 宽止盈 + 睡觉让奔跑」仅限美股早盘（北京晚上）用户睡觉时段**；**港股盘中（北京白天，用户与 AI 同步盯盘）不用此法**——止盈止损**时时刻刻据盘面主动动态调整**，不睡觉。⚠️ 「让利润奔跑」的终点是**当日停止盯盘前**（美股用户喊停或 16:00 收盘、港股 16:00 收盘；持仓不过夜、不跨交易日）——宽止盈位要在当日停止盯盘前可达，到不了就在停盯前主动按市价平掉，故宽止盈不能放到停止盯盘之后才到的位置。**锁利位是浮盈中持续重估的变量，不是一次定死**。
- **主动平仓不等固定止盈（2026-07-06 用户教训，最高优先级）**：浮盈中若**情况不对**（动能减弱/资金流边际转正/价量背离/震荡不前/滞涨），**随时 LO 主动平仓锁利**，**绝不死等预设止盈位（如 2 倍赔率）**。用户原话："主动平仓不一定要等到两倍赔率，情况不对随时平仓。" 移动止损只是被动兜底，主动判断离场才是锁利主手段——执着等固定止盈会把已到手的利润还给市场。第一单做空 MU 浮盈 +$212、近5分钟资金流一度转正 +987（动能减弱信号）时未主动平仓，后反弹变亏损 −$92，典型错过。
- **临近停止盯盘的分情况策略（2026-07-19 修订：撤销留单——停盯前一律主动平；原「持有到强平让利润跑尽」「留单赌午盘」整条作废）**：临近当日停止盯盘时点——港股 12:00 午休前 / 港股 16:00 收盘前 / 美股用户喊停或 16:00 收盘前——**停盯前必须主动平仓（不存在留单）**。主动平仓的时机判据：**有持仓**——①**「情况不对」信号**（任一即立即主动平仓锁利）：滞涨 / 动能减弱 / 价量背离 / **加速赶极端（做多持仓：加速赶顶 = 加速上涨 + 破前高 + 远离 VWAP 上方超买；做空持仓：加速赶底 = 加速下跌 + 破前低 + 远离 VWAP 下方超卖——两者原理对称反向、都是动能将竭，属「情况不对」、不是「持续上涨/下跌无滞」）**——剩余时间不多，收益一旦跌下去没足够时间涨回，此时主动平锁利；②**无情况不对信号、趋势仍健康**：可持有到停止盯盘时点前再主动平（让利润多跑一段），但**到停止盯盘时点必须主动平掉、不留单**（2026-07-19 撤销留单策略后，不再有「留单靠条件单兜底」或「持有到账户强平」的选项——停盯前主动平是唯一出路）。2026-07-17 MU 反例（旧规则下的教训，新规则下语境变了）：当时 12:00 停盯、浮盈 +$3071、加速赶顶（11:54-12:00 加速 873→884、破前高 885.93、远离 VWAP 845.70 达 +4.5%）却误判"持续上涨无滞涨"留单赌午盘 → 15:15 破 866 止损仅 +$1060、回吐 ~$2000；**新规则下教训转为**：盘中任何时候出现「加速赶极端 = 动能将竭」就主动平锁利、不等停盯时点——因为现在不存在「留单赌后续」的选项，动能将竭时锁利就是最优。**无持仓**——临近停止盯盘时点开仓要小心：剩余时间少意味着日内可达涨幅有限、止盈空间被压缩，赔率可能凑不到 ≥1.5，开仓前必须重算赔率、不足就不开。一句话：**临近停止盯盘，有仓看「动能是否将竭」决定何时主动平（将竭立即平、仍健康可持有到停盯前），无仓看「剩余时间够不够凑出赔率」决定开不开**。
- **回踩不破前低 = 最佳平仓时刻（2026-07-06 用户）**：做空时价格**回踩前低不破**（= 下跌动能衰竭、买盘承接站住）→ 这才是技术面上最佳主动平仓时刻，优先于固定止盈位；做多时回踩前高/支撑不破 = 加仓或持有时刻。错过这个点位往往由盈转亏。
- **移动止损总原则**：持仓全程移动止损线；当**止损价移过成交价到盈利方向**（做多止损价 > 成交价 / 做空止损价 < 成交价）时，这笔的胜负已锁定（最差不亏），之后让利润奔跑、止盈价朝盈利更深处调。
- **资金流入但价格不涨 = 卖盘更强（2026-07-07 港股复盘，最高优先级）**：资金流（capital inflow）只有伴随**价格同步突破**才是多头确认。若资金持续流入但价格多次冲阻力不过（价量背离），说明上方卖盘在出货、把买单全吃了——这是**离场 / 观望信号，不是入场信号**。2026-07-07 美团第二笔误把「资金流入」当成「突破在即」，七次冲 78.50 失败，亏 −HK$125。
- **阻力三次不过不赌突破（2026-07-07 港股复盘）**：一个阻力被测试 ≥3 次都不破 = 强阻力确认，下次突破概率随测试次数**下降**（不是上升，每次失败都在验证卖盘强度）。想做多就等「真放量突破站稳 + 回踩阻力转支撑不破」，不要在阻力下方提前埋伏赌突破。
- **大单吸筹 + 缓慢震荡上涨时不过早平（2026-07-20 立）**：super 超大单持续大幅净买 + 价格**缓慢震荡上涨**时，super 短期波动（停止活动 / 略降）≠ 反弹结束，不要一见 super 略降就判「动能衰竭」平仓。**离场判据**（任一即可）：① super **真正持续转流出**（大单出货）；② 价格**跌破关键支撑**（如当日低点）；③ **阻力多次不过 + 量缩**（动能真衰竭组合）。super 持平 / 略降但未转流出时倾向持有（让大单吸筹的后续拉升兑现）。⚠️ **场景限定——按上涨速度区分**：本条只适用**缓慢震荡上涨**（持有）；若标的处于**急速上涨（加速赶顶）**则应提早平、不抓顶部（信号模式执行延迟 + 急速反转，见「交易策略纪律 · 临近停止盯盘的分情况策略」加速赶极端条），不能套用本条「持有等 super 转流出」。
- **三角形持续形态未破时持有 / 移损观望、不因遇阻就平（2026-07-20 立，细化上方「阻力三次不过不赌突破」）**：三角形持续形态**未破**时，即使阻力 / 支撑多次遇阻也应**持有 / 移损观望**，不因遇阻就平——**阻力 / 支撑多次遇阻 ≠ 反向信号**，要结合形态斜率判断：遇阻 + **形态破**（支撑 / 阻力线斜率反转）才反向；遇阻 + **形态未破** → 持有 / 移损观望、等突破。形态判据（做多 / 做空对称）：
  - **上升三角形（做多持仓）**：底部支撑线正斜率（低点抬高）+ 顶部阻力线走平（高点不创新高但不回落）+ 夹角向上收敛 → 僵持后大概率**向上突破**。
  - **下降三角形（做空持仓）**：顶部阻力线负斜率（高点降低）+ 底部支撑线走平（低点不再更低）+ 夹角向下收敛 → 僵持后大概率**向下突破**。
  - **怎么用**：形态未破时 ① 不平仓；② 移损跟随形态（做多止损跟随支撑抬升、做空跟随阻力下移）；③ 观望等突破。突破后的急速段按「加速赶极端」提早锁利、不死等；反向回落破形态则移损止损出。⚠️ 本条细化「阻力三次不过不赌突破」——单看阻力次数不够，要结合形态斜率：**形态未破时的遇阻 = 持有信号，不是反向信号**。
- **做多合格入场只有两种（2026-07-07 固化）**：① 回踩支撑企稳反弹（如 2026-07-07 美团 77.50 支撑企稳，+HK$990）；② 突破后回踩阻力转支撑不破。「赌突破」不在其中——前两者依据**已发生的事实**，后者依据**预测**。
- **转势形态：头肩 / 双底双顶（2026-07-18 用户教，做多/做空入场形态判据）**：核心原理——**打破下跌趋势（高点更低 + 低点更低）= 转多信号；打破上涨趋势（高点更高 + 低点更高）= 转空信号**。形态分有头 / 无头两类，原理一致：
  - **头肩底（做多，有头）**：下跌趋势中「左肩低点 + 头（更低低点）+ 右肩低点与左肩接近持平」+ **破颈线**（左右肩之间的短期高点）→ 做多信号。
  - **双底（做多，无头）**：两个接近持平的低点（中间无更低低点）+ 破颈线 → 做多信号（"低点不再更低"= 趋势被打破）。
  - **头肩顶（做空，有头）**：上涨趋势中「左肩高点 + 头（更高高点）+ 右肩高点与左肩接近持平」+ **跌破颈线**（左右肩之间的短期低点）→ 做空信号。
  - **双顶（做空，无头）**：两个接近持平的高点 + 跌破颈线 → 做空信号。
  - **关键**：用 **close 价**判"持平"（高低点波动大、close 稳）；"持平"= 趋势被打破的核心信号。
  - **价值**：比"连破 2-3 道阻力/支撑"更早确认转势——形态在「右肩 / 第二底形成 + 破颈线」一刻即确认。2026-07-17 MU 实例：下跌段 09:37 high 824→09:39 high 815→09:45 low 804（高点低点递降）；**09:41 close 812.19（左肩）+ 09:44-45 low 804（头）+ 09:46 close 812.39（右肩与左肩持平）+ 09:47 回踩 809.79 + 09:48 破颈线 815** → 头肩底确认、做多信号；之后 MU 816→845（+29 点）。比我昨天等连破 836/840/853（11+点）早 1.5 小时。
- **量价正向确认作为开仓前置（2026-07-13 借鉴 Livermore 量价法则）**：上面「资金流入但价格不涨 = 卖盘更强」讲的是**背离**（资金进但价不涨 = 卖盘在出货，离场信号）；这里补**正向配合**（开仓确认信号）：顺势**做多要求放量**（上涨带量 = 买盘真主动）、**做空要求放量下跌或缩量反弹**（下跌带量 = 卖盘真主动）。**无量突破不可信**——价涨但量没跟上多半是诱多，不追。开仓前看一眼近几分钟量价是否配合，不配合就等。
- **不在关键点之外追单（2026-07-13 借鉴 Livermore 关键点法则）**：与「做多合格入场只有两种」呼应——那条讲**在哪入场**（回踩支撑 / 突破回踩），这条讲**为何不追远离关键位的拉伸**：价格已远离关键支撑 / 阻力位（涨了一大段没回调 / 跌了一大段没反弹）时**不追**，因为此时 EV 变差——① 离止损位远 → 仓位被迫缩小或止损被迫放松；② 拉伸越大 → 均值回归概率越高 → 胜率下降。等价格回到关键位附近（回踩 / 反抽）再进。一句话：**只在关键位附近交易，远离关键位不追**。
- **区间震荡市要切换区间交易、不执念破位做空（2026-07-08 SOXL 复盘教训，最高优先级）**：当价格在区间反复（如 SOXL 162-170 **多次假破位 / 假突破**），说明是**区间震荡**而非趋势——多次假破位本身就是「支撑有效」的信号。此时强支撑（多次测试不破）= **做多信号**（向阻力看，赔率好），强阻力 = **做空信号**（向支撑看），**而不是等破位顺势**。2026-07-08 SOXL 162 三次假破位都反弹，162 是强支撑、应在 162 做多（止损 160、止盈 170、赔率 4；实际反弹到 168 +3.7%），但我执念「板块弱做空」、反复等破 162 做空（三次假破位全错过 + 一笔保本）。教训：**≥2 次假破位/假突破 = 区间信号，立即切换区间交易（支撑买 / 阻力卖），不执念单一方向**；"二次确认避免假破位扫损"是对的，但更要从假破位读出"支撑强→可做多"。
- **半仓轮动：震荡市里替代移动止损的主动高抛低吸（2026-07-18 用户教，2026-07-18 用户纠正减半本质=降风险敞口、不止锁利）**：震荡市（价格在区间内来回反复）里，移动止损锁利不好用——价格会多次触及止损位，止损一旦上移到盈利区就被"正常的区间回调"扫掉、扫完价格又涨回去，等于卖飞在区间内的小回调上。此时改用「半仓轮动」主动降低风险敞口：**对已有持仓，在区间上沿（关键阻力位）减一半仓位、在区间下沿（关键支撑位）加回相同数量**——一来一回，最终持仓数量没变，但已经在区间上沿那一段落袋了一部分利润（震荡市持仓多为浮盈、减半表现为锁利；但减半的本质更广、与盈亏无关，见下「减半的本质」）。分方向（对称反向）：
  - **做多持仓**：价格涨到区间上沿阻力位 → 减仓一半（发 🟠 减仓信号、理由"区间上沿锁利"，减仓后剩余仓位仍按原止损管理）；价格回落到区间下沿支撑位 → 加仓等量回来（发 🔵 加仓信号、走加仓场景二用锁利额度、max_loss 与原持仓合并 ≤ max_loss）。
  - **做空持仓**：价格跌到区间下沿支撑位 → 减仓一半（🟠）；价格反弹到区间上沿阻力位 → 加仓等量回来（🔵）。
  - **为什么减"一半"而不是全平**：留一半底仓做后手——万一区间被突破转为单边趋势，底仓还能吃到顺势利润；全平则彻底卖飞。这是"半仓"轮动而非"全仓做 T"的关键，也呼应「做多合格入场只有两种」里"已发生事实"的原则：减半先锁定一半（盈则落袋利润、亏则止损一半）、降低一半后续敞口，留另一半应对判断错了的情况。
  - **减半的本质 = 降低一半风险敞口（与锁利 / 锁亏无关，看的是价格预期；2026-07-18 用户纠正）**：减半这一步的底层动机是**当价格预期不利（更糟）时、先降低一半后续风险**，**和当前是盈是亏无关**——盈利时减半 = 锁利、亏损时减半 = 锁亏，两者都只是减半降敞口后的副产物，**不是减半的目的、也不是分类开关**（纠正"减半 = 锁利"的联想）。即便当前是浮亏，只要预期价格会更糟，先减一半同样是降低一半后续风险（少亏那一半在后续下跌里的损失）——这跟锁利无关、跟价格预期有关。减完之后接不接回，看价格预期：① **预期短期能接回**（价格会回到对侧关键位）→ 打算做轮动，到对侧位加回等量、持仓数量还原；② **无法预期短期能否接回** → 先减半降风险（锁利锁亏都行、反正是降低了一半敞口），接不接回看后续演变。
  - **轮动不是死任务、接回可放弃（2026-07-18 用户立）**：即便一开始打算接回做轮动，也要实时跟踪盘面；如果走势和预想不符、到了关键位置时走势不对，可以**放弃接回动作**——让减半停留在"降风险"状态、不强行加回（走势已变时强行接回 = 加在错误位置上）。
  - **适用前提（必须先满足，否则会亏）**：① 已确认是震荡市（区间反复、≥2 次假破位/假突破，见上「区间震荡市」条）——单边趋势市误判成震荡会两头亏：上涨趋势里减半 = 卖飞后续涨幅、下跌趋势里加仓 = 越加越深，震荡市判据不过关就不准用；② 区间有清晰的阻力和支撑（高抛低吸需要明确的上下沿、上下沿定不了就无处轮动）；③ 已有持仓（半仓轮动是对已有持仓的操作，空仓时谈不上"减半"、走开仓 🟢 + 平仓 🔴 即可）。
  - **风险护栏**：① 加仓那一半必须带止损（🔵 加仓信号必含加仓止损位，独立于原持仓可更紧）——支撑被突破时新加的仓位要止损出场，不能死扛成"越跌越加"；② 加仓的 max_loss 与原持仓合并算 ≤ max_loss（属同一笔交易，见「信号格式」加仓信号场景二）。
  - **趋势市里只有波动特别大、且波动规律明显的才做轮动（2026-07-18 用户纠正适用范围）**：不是泛指"波动大的趋势行情"都能轮动——只有**波动特别大、且波动规律明显**（能预判回调 / 反弹的接回点）的趋势行情才适合轮动；规律不明显的剧烈波动无法预期接回点，减半只作"降风险敞口"处理、不强行轮动接回。趋势市轮动要顺大势（上升途中回调加、拉伸减；下降反过来），**不能逆大势高抛低吸**（逆势加仓 = 接飞刀 / 摸顶）。震荡市仍是轮动主用场（价格大概率来回、接回预期强、规律明显，见「区间震荡市」条）。

- **事后结果不反推决策对错（2026-07-07）**：评判一笔交易看「当时依据是否站得住」，不看事后盈亏。2026-07-07 第二笔事后看美团摸到 78.50，但中途 77.45 接近 77.90 止损、大概率被扫，依据（赌突破）本就不成立 → 决策错，与事后无关。
- **信号模式入场更要「已确认」而非「预测」（2026-07-07）**：用户 App 执行有延迟，赌突破 / 赌反转的标的若回落，延迟放大损失。入场点应建立在已发生的事实（企稳 / 真站稳）上，给执行延迟留容错。
- **杠杆 ETF 反弹强且久，斐波那契 0.5 回撤不是顶（2026-07-10 07709 复盘）**：2x 杠杆 ETF 超跌反弹力度大、持续久，判「反弹遇阻」不能在 0.5 回撤位（50%）就下结论，要等 0.618（61.8%）甚至 0.786 / 前高附近**多次测试不过**才算遇阻（与「阻力三次不过」同理）。2026-07-10 做空 07709（两倍做多海力士）：跌势 98.86→90.68（区间 8.18），0.5 位=94.77、0.618 位=95.74；反弹到 94.6（0.5 位）我就喊遇阻，实际穿透 0.5、0.618 一路到 **97.50（83% 回撤）**。过早判遇阻 → 踏空或被扫。**回撤位算法**：位 = 低点 +（高点−低点）×比例（0.5 / 0.618 / 0.786）。
- **止损位与入场点是 EV 综合权衡，不是「止损越远越安全」（2026-07-10 07709 复盘）**：做空时止损近=赔率高但胜率低（易被反弹扫），止损远=胜率高但赔率低。必须按 EV=胜率×赔率−败率×1 综合判，不能只看赔率或只看「不被扫」。2026-07-10 07709 在 94 做空：止损 96 赔率 2.5 但胜率≈0（午后 97.5 必扫，EV=−1）；止损 99 胜率 85% 但赔率仅 1.0（EV=+0.7 薄）——差入场点上怎么选止损都别扭。**正解是等更好的入场点**：午后 97.5 遇阻做空、止损 99、赔率 5.7、胜率 80%（EV=+4.36），「止损近且不被扫」与「赔率高」同时成立。核心：**找高胜率 + 高赔率同时成立的入场点，而非在差位置纠结止损远近**。
- **领先指标持续走强时不做空主标的（2026-07-13 SOXL 实盘复盘，亏 $81.7 / HK$637）**：做空 ETF 时，其重仓成分 / 领先指标（如 SOXL 之于 MU/SNDK）是方向驱动。**领先指标持续创新高时，ETF 阻力即使"三次不过"确认也会被突破**——领先指标强势是破阻力的动能，机械套用「阻力三次不过」会吃亏。2026-07-13 实盘做空 SOXL 40 股 @ 171.82、止损 173.5：SOXL 173 阻力三次不过、MU 冲 932.99 回落 929（看似失败新高）时入场，但 MU 随后继续创新高 936.47，带 SOXL **持仓仅 50 秒**即冲破 173.5 止损，亏 $81.7。教训：① **领先指标持续创新高时不做空主标的**，等领先指标明确见顶（连续 ≥2-3 轮不创新高 + 买卖比持续负）再空——这条优先级高于 ETF 自身的阻力确认；② **「失败新高」判断要严**：单次冲高回落 ≠ 见顶，需连续多轮确认（与「杠杆 ETF 反弹强且久」同理，反弹 / 强势持续力易被低估）；③ **3x ETF 急涨时市价止损有滑点**（本单 173.5→173.8625，+0.36 / +0.21%），止损位预留滑点 buffer、或 max_loss 预算计入滑点（本单超 $80 预算即因滑点）；④ **纪律一致性**：盯盘中前几轮因"MU 强不空"反复撤信号，这一轮 MU 仍强（932）却破例空——临场不为"想开仓"而放宽自己立的标准。
  - **【2026-07-13 第二单升级，累计两单亏 HK$899】**同日第二单做空 SOXL 30 股 @ 173.41、止损 174.5：**已遵守①**（等 MU 连续 4 轮不破 940.45 才空），但 MU **再次创新高 943.50**、SOXL 33 秒涨破止损，亏 $33.6。教训升级：① **「连续 N 轮不创新高」也不是见顶充分条件**——强反弹市领先指标震荡整理后会再突破，"不创新高 N 轮"只说明短期整理、不说明趋势反转；见顶须确认**明确转跌**（连续阴线 + 跌破前回调低点），非仅"不创新高"；② **强反弹市（暴跌后 V 型反弹）不做空反弹**——反弹力度反复超预期（SOXL 165.6→177.66 +7.3%），做空反弹策略本身在高波动反弹日失效、反复被扫；③ **做空等下跌趋势确认**（标的破前低 / 关键支撑、反弹明确结束），不抢反弹顶部（抢顶 = 赌反转、胜率低）；④ **上涨途中做空执行价变差**：信号发出 SOXL 171.57 → 用户执行 173.41（3 分钟续涨 1.84），反弹中做空越拖越不利（与「信号时效性」总则呼应，但此处延迟来自执行端 + 价格本身续涨，非 AI 反复 quote）。
  - **【2026-07-14 次日复盘修正——方向其实对，核心是 3x 止损太近】**次日查 SOXL 7/13 完整 K 线：开 174.61 / 高 177.67 / **低 161.55** / 收 165.37——SOXL 下午确实大跌、破前低 165.6 到 161.55，**两单做空方向完全正确**。若不止损持有到收盘 / 最低，本可盈利 $499-766（HK$3890-5974），实际亏 $115（HK$899）全因止损被盘中反弹扫。**核心失误是止损太近（用户纠正）**：单1 止损距离 0.98%、单2 0.63%，而 SOXL 7/13 日内振幅 9.23%——止损只有振幅 1/9~1/15。**教训（用户立）：3x 杠杆 ETF 止损要远**——3x 日内正常波动 3-5%，止损至少给 **3-5% 距离**（振幅 1/3-1/2），否则方向对也必被盘中反弹噪音扫、拿不住。⚠️ 上方②「强反弹市不做空反弹」需修正为：**做空反弹方向可对，关键是 ①3x 止损给足波动空间（3-5%）②入场等反弹结束 / 破位确认、不在反弹途中空 ③转破位空后要耐心等破位（本日收工太早 11:51 美东，错过下午跌破 165.6 到 161.55 的破位机会）**。

1. **方向大于风控**：逆势交易风控再细也难赚。建仓前必看近 5 日 K 线趋势，只在上升趋势回踩或下降趋势反抽时顺势入场，不接飞刀。
2. **资金流 > 盘口 > 分时**：盘口墙可造假（挂大单压价/闪挂闪撤），但持续的大单净流入要真金白银、难造假。**盯盘看资金流默认走富途 `get_capital_flow`**（含 super 超大单 + 分钟序列，比长桥更细，2026-07-15 实测立，见「数据源分工总则」）。盘口偏空但大单持续净流入 → 倾向"洗盘"持有；盘口强但大单净流转负 → 警惕"对倒出货"。判别假/真盘口墙：连续 2-3 轮采样（每轮 30-50 秒）观察是否"持久挂单 + 价格真涨" vs "闪挂闪撤 + 价格不动"。
3. **加仓要等回踩不破支撑**，不追在突破点。强势突破不给回踩则放弃加仓，宁错过不追高。
4. **没有明确顺势信号时宁可空仓**，不为扳回亏损强行交易（"最大化收益率 ≠ 最大化交易次数"）。
5. **连亏不是停手理由，信号不合格才是**：AI 无情绪干扰，两连亏后不带"扳回"执念、仍按"顺势 + 资金流 + 主动止盈"纪律执行，有正期望（2026-07-03 前两单逆势亏 -301，后两单顺势 +66 扳回 22%）。
6. **找标的必须先过热度榜（2026-07-07 用户铁律，覆盖旧"优先"措辞）**：凡是用户让"找 / 选 / 筛交易标的"或问"有什么可做的 / 看看机会"，**第一步必须跑富途 `get_hot_list(market)` 热度榜**筛选，**不许凭下方关注列表或 AI 记忆直接定标的**。流程：热度榜头部 → 叠成交额 / `volume_ratio` 筛流动性 → 叠顺势过滤（近 5 日趋势）→ `static` + `classify_hk_security.py` 核实代码与类型 → 才进入分析（方法细节见下方子条「热度判断方法」）。原 2026-07-06「优先近期热点」升级为硬性必须。**备选关注列表（仅作热度榜之外的参考，不作直接选标的依据；会变，下单前必须核实）**：半导体存储链——海力士 / 三星电子杠杆 ETF、美光科技 MU。⚠️ **杠杆/反向 ETF（2x）合规待用户确认**：形态是 ETF、但内部用衍生品实现杠杆，与护栏⑦「禁衍生品」有张力，交易前需用户明确是否纳入允许范围 → **已确认允许（2026-07-06，2026-07-09 扩展含 3x 如 SOXL）**，下单前核实代码 + 流动性即可。
   - **热度判断方法（2026-07-06 实测）**：富途 `get_hot_list(market)`（热度榜，关注度+资金维度）比 `get_top_movers_rank`（纯涨幅榜）更靠谱；选热度标的必须叠 `volume_ratio`/成交额筛流动性，排除「振幅高但盘口薄」的小盘题材（实测港股涨幅榜 02526 振幅 283% 那种 = 陷阱区）。热度榜头部 + 流动性好 + 顺势 = 高效机会。实测美股热度榜头部为半导体存储链 MU/SNDK/NVDA/SOXL。

## 盯盘节奏

**盯盘持续性（2026-07-13 用户铁律，2026-07-16 补充启动与停止边界，2026-07-19 修订美股停止边界）**：**AI 不自主启动盯盘，收到用户的盯盘指令才开始**；启动后**未叫停就持续盯，不中途停下等用户**。靠 cron 循环（**固定每分钟触发**、session-only）+ 密采样维持，任务不因单轮对话结束而终止——无信号时一行带过继续盯，不暂停问"要不要继续"。方向研判、找标的、盯信号、发信号、平仓全程连续推进，直到**用户喊停、或撞上强制终止盯盘边界**为止。港股两个强制边界（见「时点平仓与停止盯盘约束」）：港股 12:00 午休前（主动平仓 + 终止盯盘，13:00 午市不自动恢复、等用户指令）、港股 16:00 收盘前（主动平仓 + 终止盯盘）。美股不设固定终止时点——盯到**用户发出停止盯盘指令**（大概率北京晚上睡觉前）或**美东 16:00 收盘**（取先到者），停前主动平仓、不留单。

### 盯盘启动方向研判（盯盘第一步定当日基调，2026-07-13 借鉴 OhMyTradingTeam 多视角框架日内化）

盘中找标的（hot_list）是"看今天什么在动"，但缺一步先定"今天大方向偏多还是偏空"。本研判用**多维度独立打分后综合**，定当日方向倾向，缩小找标的范围（只做顺势方向）——避免见涨追涨、逆势开仓。

**时机**：盯盘启动的**第一步**，在 `preflight` 之后、`hot_list` 之前——**无论用户何时叫醒盯盘（盘前预热或已盘中）都先跑一次**，产出：一句话"今日港股/美股偏多 / 偏空 / 中性震荡"。盘前启动用隔夜外围 + 指数结构判基调；盘中启动数据更全（有实时资金流 / 量价），照样走这五维度。核心是"**先定方向再找标的**"，时点不限、不是只在盘前做。

**五个维度独立给倾向**（每维偏多 +1 / 中性 0 / 偏空 −1，最后求和定向）：

| 维度 | 看什么 | 来源 |
|---|---|---|
| **宏观基调** | **最近一根**美股日 K 收盘涨跌 / 美 10 年期国债收益率方向 / 美元指数 / VIX；今日有无央议息、CPI/非农、重大政策 | 富途 `get_market_snapshot` 查美股主指数 / ETF；外围消息看新闻。⚠️ **"最近一根"≠"昨夜"**：周一开盘时最近一根是**上周五**（周末美股休市），节假日后开盘同理跨假期——须跨周末/假期核对期间事件（地缘/政策/数据），不许说"昨夜"、不许忽略中间跨度（2026-07-13 用户纠正） |
| **技术趋势** | 目标市场指数近 5 日 K 线结构（上升 / 下降 / 横盘）、距关键支撑阻力远近 | `scripts/kline.py <指数代码> 5` |
| **量价** | 指数近几日涨跌是否带量（量价配合 = 趋势健康，背离 = 虚弱） | kline 量柱 |
| **资金流** | 目标板块 / 标的资金净流向（盘前数据有限，开盘后用 `capital` 复核） | 富途 `get_capital_flow`（盘中）|
| **事件催化** | 当日有无个股 / 板块级事件（财报、产品发布、政策落地）——事件日波动大、方向不定 | 财经日历 |

**综合定向**：
- 和 ≥ +2 → 今日偏多，找标的优先找顺势做多机会（hot_list 后叠"近 5 日上升趋势"过滤）。
- 和 ≤ −2 → 今日偏空，优先找顺势做空机会。
- 其余 → 中性震荡，按区间交易思路（支撑买 / 阻力卖，见「交易策略纪律」区间震荡条），降低开仓预期。

**⚠️ 位置评估：当前价在大趋势中的位置 = 盘中方向假设的关键**（2026-07-18 用户立，优先级高于五维度求和定向）。五维度求和给的是"大趋势方向"（偏多/偏空），但日内盘中走哪边还要看"**当前价在大趋势中的位置**"——同一支下跌趋势股，处在上沿（超买）和处在下沿（超跌）的盘中方向完全不同：

| 当前价位置 | 盘中倾向假设 | 即使大趋势 |
|---|---|---|
| **下沿（超跌区）**：接近最近 ≥4 个月日线低点连线 / 重要支撑 / 短期大幅超跌 | **超跌反弹做多假设**（等企稳做多、不空）| 即便大趋势空，下沿超跌最可能深 V 反弹 |
| **上沿（超买区）**：接近最近 ≥4 个月日线高点连线 / 重要阻力 / 短期大幅超买 | **回调做空假设**（等遇阻做空、不多）| 即便大趋势多，上沿超买最可能回调 |
| **中段** | **顺势大趋势方向** | 下跌中段续空、上涨中段续多 |

**怎么判断位置**：盯盘启动时 `python3 scripts/kline.py <标的> 120`（如 `US.MU 120`，约 4 个月交易日；**判断大趋势至少拉 4 个月日线**——短周期看不出真趋势通道，2026-07-18 用户立）画高点连线（上沿阻力）+ 低点连线（下沿支撑），开盘价/现价接近哪沿即位置判据；**大趋势方向 × 位置 = 今日盯盘方向假设**（例：大趋势空 + 下沿超跌 → 超跌反弹做多假设；大趋势空 + 中段 → 顺势做空假设）。

**重要边界**（避免误用）：
1. **本研判是"大方向参考 + 选标的过滤"，不是盘中信号**。方向偏多不代表立即追多，仍要等盘中合格入场点（回踩企稳 / 真突破站稳，见「做多合格入场只有两种」）。
2. **宏观基调维度较慢**，只影响"今日整体偏多偏空"的背景判断，不进实时盯盘链路（盯盘看秒级盘口，宏观是背景）。
3. **事件催化日**（CPI / 非农 / FOMC / 财报）方向不定但波动剧：优先上 WebSocket 密采样（见下「重要数据日优先 WebSocket」），开盘后等波动消化再入场、不抢第一波。
4. 中性震荡日**不是不开仓**，而是切区间交易（支撑买 / 阻力卖）、降单笔预期、止损更紧。
5. **⚠️ 盘中动态修正方向（2026-07-17 教训立，最高优先级）**：开盘方向研判是"起点"不是"全天锚定"——盘中必须随趋势演变动态校准，方向变了就切换策略，**绝不固守开盘方向执念**。反转触发条件（任一满足即重估方向）：① **连续突破 ≥2 道关键阻力**（创新高）= 趋势转多信号，不是"反弹遇阻"——放弃做空框架、转向做多/观望；② 连续跌破 ≥2 道关键支撑（创新低）= 转空，放弃做多；③ 标的站上/跌破 VWAP 并企稳 = 日内多空转换。**"反弹"vs"反转"判定**：连续突破多道阻力 = 反转/趋势性上涨（反弹只是短暂回升）；连破 2 道阻力即应重估方向。2026-07-17 教训：开盘偏空后 MU 连破 836/840/853/860 四道阻力（明确转多），却执念"等反弹遇阻做空"2 小时全踏空、靠用户提示才修正为做多（错过 804→880 整段）。**每隔 15-30 分钟、或标的突破关键位时，重跑一次方向研判**（看趋势/量价/资金流是否仍与开盘方向一致），不重估 = 把开盘研判当信仰。

**样本定义（T）**：**每次平仓/减仓结算都算一个 T（一个样本）**——不管是主动平仓 🔴、减仓 🟠（部分平仓）、移动止损 🟡 触发被动平仓，**每一次仓位了结都计为一个独立 T**。一个开仓若分多次了结（如先减仓一半、再止损平剩余），每次各算一个 T、各按对应股数算盈亏。⚠️ **不存在「未平仓」的样本**——「持仓不过夜」纪律下，所有持仓在当日停止盯盘前（港股 12:00 午休前 / 16:00 收盘前、美股用户喊停或 16:00 收盘前）由 AI 主动平掉，每个开仓都会被了结。N = 平仓事件数。

**统计量定义与推导**（Python 实算各笔 $P_i$ 代入下列公式，不心算）

*Notation.* 已了结样本集 $\mathcal{T}=\{T_1,\ldots,T_N\}$，$N=|\mathcal{T}|$ 为样本总量（= 平仓事件数）。第 $i$ 笔实际盈亏 $P_i\in\mathbb{R}$（$P_i>0$ 盈利、$P_i<0$ 亏损）。单笔最大损失额度 $M:=\text{max\_loss}>0$（各笔相同）。

*Definition（R-multiple）.* $R_i:=P_i/M$。由止损反推仓位（股数 $=M/\Delta_{\text{stop}}$，$\Delta_{\text{stop}}$ 为止损距离）保证 $P_i\geq -M$，故 $R_i\in[-1,+\infty)$。

划分 $\mathcal{W}:=\{i:P_i>0\}$、$\mathcal{L}:=\{i:P_i<0\}$，$N_W:=|\mathcal{W}|$、$N_L:=|\mathcal{L}|$，$N=N_W+N_L$。

*Definition（胜率 / 败率 / 胜赔率 / 败赔率）.*
$$p:=\frac{N_W}{N},\quad q:=\frac{N_L}{N}=1-p,\quad R_W:=\frac{1}{M\,N_W}\sum_{i\in\mathcal{W}}P_i\;(>0),\quad R_L:=\frac{1}{M\,N_L}\sum_{i\in\mathcal{L}}P_i\;(\in[-1,0))$$

*Proposition（EV 分解—合并恒等）.* 定义期望收益 $\mathrm{EV}:=\dfrac{1}{N}\sum_{i=1}^{N}R_i=\dfrac{1}{M\,N}\sum_{i=1}^{N}P_i$，则
$$\mathrm{EV}=p\,R_W+q\,R_L.$$

*Proof.*
$$\begin{aligned}
pR_W+qR_L
&=\frac{N_W}{N}\cdot\frac{1}{M\,N_W}\sum_{i\in\mathcal{W}}P_i+\frac{N_L}{N}\cdot\frac{1}{M\,N_L}\sum_{i\in\mathcal{L}}P_i\\
&=\frac{1}{M\,N}\!\left(\sum_{i\in\mathcal{W}}P_i+\sum_{i\in\mathcal{L}}P_i\right)\\
&=\frac{1}{M\,N}\sum_{i=1}^{N}P_i=\mathrm{EV}.\qquad\blacksquare
\end{aligned}$$

*Corollary（预估 EV）.* 开仓前取预估胜赔率 $\hat{R}_W\geq 1.5$、保守取 $R_L=-1$（假设每次亏满 $M$），则 $\mathrm{EV}_{\text{est}}=p\,\hat{R}_W-q$，即「交易策略纪律」的 $EV=\text{胜率}\times\text{赔率}-\text{败率}$。

*Definition（百分制 / 金额制）.* $\mathrm{EV}\%:=100\,\mathrm{EV}$；$\overline{P}:=\dfrac{1}{N}\sum_{i=1}^{N}P_i=M\cdot\mathrm{EV}$（平均每单盈利金额）。

*Definition（过程指标 MAE / MFE / 回吐 / 锁利效率，2026-07-21 立）.* 终局盈亏 $R_i$ 是滞后指标（要等平仓、且样本效率低）；持仓过程指标逐笔即可观测、信息量大得多，专门用于样本不足时早期暴露问题。记第 $i$ 笔持仓期间（开仓时刻 → 平仓时刻）按持仓方向的最大不利偏移浮亏、最大有利偏移浮盈，各自除以 $M_i$ 归一化到 R 单位：

$$\mathrm{MAE}_i\in[-1,0],\qquad \mathrm{MFE}_i\in[0,+\infty).$$

（做多：$\mathrm{MAE}_i$ 取期间最低价、$\mathrm{MFE}_i$ 取期间最高价，相对开仓价；做空方向互换。两者均为对应浮亏 / 浮盈峰值金额 $\div M_i$。）

$$\text{回吐}_i:=\max(\mathrm{MFE}_i-R_i,\ 0)\geq 0.$$

> 注：$\mathrm{MFE}_i$ 是持仓期间浮盈峰值、$R_i$ 是平仓兑现值（必然落在峰值之内），故 $\mathrm{MFE}_i\geq R_i$ 恒成立，$\max(\cdot,0)$ 是防御数据异常（跳空 / 滑点 / 采样缺失）的冗余写法、正常情况不改变取值。

*Definition（锁利效率，仅盈利单）.* $\eta_i:=R_i/\mathrm{MFE}_i$（仅当 $\mathrm{MFE}_i>0$ 有意义），含义为实际兑现占峰值浮盈的比例，$\eta_i\to 1$ 锁利越充分。**仅盈利单（$R_i>0$）时 $\eta_i\in(0,1]$ 为有效正值指标；亏损单 $R_i<0$ 时 $\eta_i<0$ 不具「锁利」语义**，故锁利效率只在盈利单子集上统计。

**分盈亏单报告（关键，避免语义混淆）**：盈利单的「回吐」= 锁利回吐（行情给了机会没抓住，改移动止损 / 止盈）；亏损单的「回吐」= 盈亏落差（由盈转亏 / 止损距离，改入场质量 / 止损时机）。两者改进方向不同，混在一起平均会让指标解读变脏。故过程指标**分盈利单子集 $\mathcal{W}$、亏损单子集 $\mathcal{L}$ 各报一组**：

| 子集 | 报告项 |
|---|---|
| 盈利单 $\mathcal{W}$ | $\overline{\mathrm{MAE}}_{\mathcal{W}},\ \overline{\mathrm{MFE}}_{\mathcal{W}},\ \overline{\text{回吐}}_{\mathcal{W}},\ \overline{\eta}_{\mathcal{W}}$ |
| 亏损单 $\mathcal{L}$ | $\overline{\mathrm{MAE}}_{\mathcal{L}},\ \overline{\mathrm{MFE}}_{\mathcal{L}},\ \overline{\text{回吐}}_{\mathcal{L}}$ |

三者合一个完整故事：$\overline{\mathrm{MAE}}$ 看防守（浮亏峰值，越接近 0 越好）、$\overline{\mathrm{MFE}}$ 看进攻（行情给的机会，越大越好）、$\overline{\text{回吐}}$ 看出场效率（没拿住的浮盈，越小越好）、$\overline{\eta}$ 看锁利充分度（盈利单，越接近 1 越好）。

**⚠️ 数据约束**：算 $\mathrm{MAE}/\mathrm{MFE}$ 需要每笔持仓期间的最高 / 最低价，而现有 signals 记录只有开仓价 + 平仓价两个端点、没有期间 high/low。两种落地：（a）复盘时按开 / 平仓时间戳（均精确到秒）回拉富途历史分钟 K，取该时间窗 max(high)/min(low)——历史样本可补算；（b）**推荐**——以后每笔平仓记录原生多写 `mfe_R`/`mae_R` 两个字段（盯盘时本就在采样价格、零额外成本），复盘直接读、不必每次回拉。历史 4 笔用（a）回补。

**复盘时逐项报告**：$N$（样本总量）；$p,\,q$（胜率 / 败率）；$R_W,\,R_L$（胜赔率 / 败赔率）；$\mathrm{EV}=pR_W+qR_L$ 及 $\mathrm{EV}\%$；$\overline{P}$（平均每单盈利金额）；**过程指标分盈亏单各报一组**（$\overline{\mathrm{MAE}},\ \overline{\mathrm{MFE}},\ \overline{\text{回吐}}$，盈利单加报 $\overline{\eta}$）；用户临时指定的其它分析。

**输出**：表格呈现各项指标 + 简短解读（EV 正负、胜率赔率是否支撑正期望、哪些笔拖后腿 / 哪些笔正贡献）。⚠️ 复盘盈亏按信号记录的参考价与 max_loss 实算，**不涉及真实账户资金**（信号模式：AI 不管账户）。

## 样本量规划：要多少笔才可信（2026-07-21 立）

胜率 / EV 等终局统计量是**渐进结论**——大数定律只保证 $N\to\infty$ 时样本均值收敛到期望，**不回答「第几笔收敛到多准」**（大数定律给方向、中心极限定理给刻度）。要回答「多少笔够」，用中心极限定理 + 置信区间 / 功效分析。⚠️ **所有公式的方差都用当前样本估的标准差 $s$ 代入——故样本量需求随真实参数变、且小样本估的 $s$ 本身高方差，这只是「用当前最粗估计做的初步规划」、不是定论。**

三个公式（95% 置信取 $z=1.96$）：

1. **胜率 $p$（比例）**：$\displaystyle n=z^{2}\,p(1-p)/d^{2}$，$d$ 为胜率绝对误差。$p=0.5$ 时方差最大、最保守（实测胜率常接近 0.5，故默认按 0.5 估）。
2. **EV 估计精度（均值 CI 半宽）**：$\displaystyle n=z^{2}\,s^{2}/d^{2}$，$s=\mathrm{std}(R_i)$，$d$ 为 EV 绝对误差（R 单位）。
3. **确认 $\mathrm{EV}>0$（单样本单侧 $t$ 检验功效，$\alpha=0.05$）**：$\displaystyle n=(z_{\alpha}+z_{\beta})^{2}\,s^{2}/\mu^{2}$，$\mu$ 为假设的真实 EV、$1-\beta$ 为把握（80% → $z_\beta=0.84$、90% → $1.28$）。正态近似、小样本偏乐观。

**读法示例（N=4 快照，$s=0.638$，仅示范如何读、不是定论）**：

| 指标 | 误差 / 假设 | 需要样本 |
|---|---|---:|
| 胜率（$p=0.5$） | ±5% | 384 笔 |
| 胜率（$p=0.5$） | ±3% | 1067 笔 |
| EV 估计 | ±0.2R | 39 笔 |
| EV 估计 | ±0.1R | 156 笔 |
| 确认 EV>0（80% 把握） | 真实 EV=0.05R | 1006 笔 |
| 确认 EV>0（80% 把握） | 真实 EV=0.10R | 252 笔 |
| 确认 EV>0（80% 把握） | 真实 EV=0.20R | 63 笔 |

核心规律：**EV 越薄（越接近打平），确认它 $>0$ 需要的样本越多（与 $\mu$ 平方反比）**。若策略真有 0.20R 的厚期望，约 63 笔可 80% 把握确认；若只有 0.05R，要上千笔。⚠️ **鸡生蛋提醒**：上表数字基于 $N=4$ 估的 $s$，真实 $s$ 可能差很多、需求随之大幅变动——这只是初步规划；且当前实测 EV=+0.19 是点估计、其 95% CI 横跨 $[-0.43,+0.82]$ 跨过 0，**现有样本根本不足以判断策略正负**。每次复盘用当下样本重算 $s$ 后重填此表。

## 多层评估框架：样本不足时怎么评估交易水平（2026-07-21 立）

本项目是 AI 主观交易、**不可回测**（LLM 不可 100% 复现、有前视偏差，见「表现验证」节），样本只能靠盘中实战积累、时间成本高。**核心转换：从「等够样本算终局 EV」转向「逐笔榨过程 + 剥离运气」**——胜率 / EV 只衡量「结果」这一个维度且最吃样本量，而决策过程质量、纪律执行、信息利用三个维度都可逐笔即时评估、不需几百笔。类比评估棋手：不必等下满 400 盘看胜率，每一手「当时局面下是不是好棋」可即时复盘评定。四层方法，从「当笔就能用」到「长期攒样本时榨干每笔」：

### 一、即时层（每笔就能评，不等样本量）

1. **决策过程审计（评分卡）**：每笔开仓同时记「决策是否合规」——是否顺势、入场类型是否落在「回踩企稳 / 突破回踩」（而非「赌突破」）、赔率是否达标（门槛见「交易策略纪律」开仓基准与「发信号前自查清单」，不在此写死以免不同步）、止损位是否给足波动、置信度是否匹配；盘后逐笔回评。**决策违规率**（多少笔违反自己立的规矩）是比胜率更早暴露问题的指标，且可归因（纪律问题 vs 方向问题）。现有 4 笔里即有 2 笔决策瑕疵（07-07 美团#2 赌突破、07-15 07709#2 无量企稳做多）。
2. **过程指标 MAE / MFE / 回吐 / 锁利效率**：见上方 Definition——把原本要几百笔才能从 EV 读出的「锁利能力差 / 防守松」，压缩到个位数样本显形。
3. **置信度校准（calibration）**：看实际胜率是否随开仓置信度（🟢 / 🟡 / 🔴）递增。🟢 高置信实际胜率与 🟡 无差别 = AI 不知道自己何时真懂、何时在猜（元认知没校准）。有标了置信度的样本即可开始看趋势。

### 二、累积层（边攒边更新，专门为「样本贵」设计）

4. **贝叶斯更新（完整贝叶斯 NIG 共轭，σ 也估；2026-07-21 修订：弃经验贝叶斯）**：不必等凑齐固定 N 再判——给 EV 一个先验、每平一笔用 $R_i$ 更新后验，后验逐步变窄、动态读「证据更偏向正 / 负期望」。**用完整贝叶斯（正态-逆伽马 NIG 共轭，$\mu$ 与 $\sigma^2$ 联合估计），不用经验贝叶斯（$\sigma$ 固定为样本 $s$）**——后者小样本下系统性偏乐观（忽略 $\sigma$ 自身的不确定性；本项目 N=4：经验贝叶斯 $P(\mathrm{EV}>0)=71.6\%$ vs 完整贝叶斯 $67.0\%$，差 4.5pp）。

   - **模型与先验**：$R_i\sim N(\mu,\sigma^2)$，要估 $\mu=$ EV。NIG 先验 $(m_0,k_0,a_0,b_0)$ 取弱信息默认 $m_0=0$（不预设 edge）、$k_0=1,\ a_0=1,\ b_0=1$。
   - **后验参数**（观测 $R_1,\dots,R_n$、样本均值 $\bar R$、离差平方和 $S=\sum(R_i-\bar R)^2$）：

     $$m_n=\frac{k_0 m_0+n\bar R}{k_0+n},\quad k_n=k_0+n,\quad a_n=a_0+\tfrac n2,\quad b_n=b_0+\tfrac12 S+\frac{k_0 n(\bar R-m_0)^2}{2(k_0+n)}$$

   - **$\mu$ 的边缘后验为 t 分布**：$\mu\,|\,\text{data}\sim t_{2a_n}\!\left(m_n,\ \sqrt{b_n/(a_n k_n)}\right)$（自由度 $2a_n$、位置 $m_n$、尺度 $\sqrt{b_n/(a_n k_n)}$）。
   - **核心输出 $P(\mathrm{EV}>0)$**：$\displaystyle P(\mathrm{EV}>0)=1-T_{2a_n}\!\left(\frac{-m_n}{\sqrt{b_n/(a_n k_n)}}\right)$（$T_\nu$ 为自由度 $\nu$ 的 t 分布 CDF）。
   - **⚠️ 统一带区间输出（小样本、大样本一律，2026-07-21 立）**：单点 $P(\mathrm{EV}>0)$ 依赖 $\sigma$ 估计、而 $\sigma$ 小样本下极不确定（N=4 时 $\sigma$ 的 95% CI 跨 6.6 倍），**禁止只报单点**。必须配敏感性范围，格式一律：**「$P(\mathrm{EV}>0)=X\%$（$\sigma$ 不确定下 $Y\%\sim Z\%$）」**。敏感性范围算法：取 $\sigma$ 的 95% CI（卡方）$\big[\sqrt{(n-1)s^2/\chi^2_{0.975,n-1}},\ \sqrt{(n-1)s^2/\chi^2_{0.025,n-1}}\big]$，在两端各算一次后验 $P(\mathrm{EV}>0)$（近似用固定 $\sigma$ 的正态后验即可），取其范围。示例（本项目 N=4）：「$P(\mathrm{EV}>0)=67\%$（$\sigma$ 不确定下 $54\%\sim 85\%$）」。
   - **判读纪律**：小样本（敏感性跨度 >5pp 或点值近 50%）**只取方向（略偏正 / 负）、不取数值、不作加仓 / 改策略决策**；大样本（跨度 ≤5pp 且点值远离 50%）点值才有参考价值、区间整体 >95% = 确认正期望。跨度随 $N$ 按 $1/\sqrt n$ 收窄：N=4~10 不可确认、N=20~30 边界、N≈50+ 才有确认价值（与「样本量规划」一致）。**关键认知：贝叶斯不绕过样本量约束**——与频率派受同样信息量限制，小样本下 $P(\mathrm{EV}>0)$ 的「精确」是表象、底层不确定性与频率派宽 CI 等量。
   - **序贯检验 SPRT（同源可选，复盘通常不必，2026-07-21 降级）**：Wald 序贯概率比检验——边收数据边检验、$\Lambda_n$ 跨边界即判（接受 H₁ 有 edge / 接受 H₀ 无 edge / 继续）。**关键：$\Lambda_n$ 本质就是对数贝叶斯因子，与上方贝叶斯 P(EV>0) 同源、信息重叠**——「SPRT 接受 H₁」≈「贝叶斯 P(EV>0) > 阈值」，未给出贝叶斯后验之外的信息。SPRT 仅有的独有价值是**明确的停止规则 + 频率派错误率保证（α、β）**，只在需要「自动判决程序 / 策略上线 gate（攒够证据才放量）」时才用得上；纯复盘评估用贝叶斯 P(EV>0)（带敏感性区间）即可，**SPRT 不必另算**。本项目样本被动慢积累、SPRT「早停省样本」红利有限，故复盘脚本 `review.py` 不实现 SPRT。
5. **按情境分类合并**：总样本少，但按方向 / 入场类型 / 标的类型 / 市场 / 置信度分组后，同类样本合并更早收敛。「赌突破类反复亏」这类模式，十来笔就能暴露——要的不是 400 笔混合样本，而是「哪类情境下反复出错」。

### 三、对照层（剥离运气，回答「盈亏里多少是 AI 的功劳」）

6. **反事实基准**：对每个开仓信号事后算三个对照——随机开仓同标的（随机基准）、从开仓点到平仓点标的本身涨跌（买入持有基准）、反向做（反向基准）。连续多笔「反向更赚」= 方向判断系统性反了。**基准跑赢占比**是比胜率更干净的「水平」度量。
7. **量化策略基准**：见「表现验证」节——写一套可回测的纯 K 线量化策略（5 日顺势 + 放量突破回踩等），回测标定可信度。它同时也是评估 AI 水平的标尺：AI 信号 vs 量化信号的一致率、分歧时谁对，直接量化 AI 相对「可形式化策略」的增量价值（K 线数据富途有 5.5 年分钟级、回测零成本）。

### 四、覆盖层（保证结论能泛化）

8. **刻意覆盖多市场环境**：小样本最大风险是偏在单一环境（如 4 笔全是大跌反弹日做多）。主动确保样本覆盖趋势 / 震荡 / 大涨 / 暴跌 / 数据日等多种环境——环境多样性比样本数本身更影响结论可靠度。

**诚实边界**：以上方法的作用是「样本不足时榨干每笔信息、尽早暴露问题、避免自欺」，**不取消对真实样本的需要**——「策略长期正期望」这个终极判断没有捷径、终归靠积累。区别只是：只看胜率 / EV 要 60–250 笔才出信号、期间两眼一抹黑；加上过程审计 + MAE/MFE + 校准 + 反事实基准，**第 5 笔就开始出可归因的信号**。
