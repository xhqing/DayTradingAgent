# 信号记录（用户下单基准 + 复盘用，港美分文件）

本目录持久化每个交易日发出的信号。**信号文件是用户下单的基准**——用户实际参照下单的是这里的内容，不是对话里的表格（对话流会把信号推走、易丢失，文件才是稳定可回查的基准）；同时供事后复盘统计（胜率 / 赔率 / EV 收益）。

**信号时序（2026-07-18 立，覆盖原「先对话、后记录」）**：决定发信号后，**第一个动作是把信号内容写入本目录的当天文件**（由 `alert.sh` 一次性完成「① 写文件 → ② 响铃 + 记录响铃时刻到 `ring-log.csv`」），**之后 AI 再在对话窗口用表格 + emoji 输出同一份内容**。文件先于响铃写好——确保用户被铃叫醒 / 吸引过去时，文件已是完整、可直接下单的内容。完整时序与 `alert.sh` 调用方式见 `../SKILL.md`「信号格式 · 🔔 信号时序总则」。

## 文件命名（2026-07-15 港美分开）

- 港股信号：`YYYY-MM-DD-HKT-signals.md`
- 美股信号：`YYYY-MM-DD-ET-signals.md`

文件名带 HKT / ET 时区标识，港股、美股信号分别记录，便于按市场分别复盘。

## 规则

- **港美分开、每天每市场一个文件**（如港股 `2026-07-15-HKT-signals.md`、美股 `2026-07-15-ET-signals.md`）。
- **先写文件、再响铃、最后对话输出**：决定发信号后第一个动作 = 调 `alert.sh` 把信号内容 append 到对应市场文件 + 响铃（脚本内顺序：① 写文件 → ② 响铃 + 写 `ring-log.csv`），之后 AI 再在对话输出同一份内容（时序见 `../SKILL.md`「信号格式 · 🔔 信号时序总则」）。
- **每条信号含**：`═══` 框线 + emoji 标题（如 `🟢🟢🟢 开仓 · <标的代码> <中文名> · 做多 🟢🟢🟢`，含发信号时间 HH:MM:SS）+ 字段表格 + 可选的结果注释。**格式与对话窗口一致**（由 `alert.sh` 写入同一份文本；表格内不放装饰 emoji、开仓 / 加仓的参考价与止损价加粗，详见 `../SKILL.md`「信号格式」）。发信号时间精确到秒。
- **记录字段**：发信号时间、标的、方向、量、参考价、止损、止盈、最大损失、置信度、依据、风险提示（**不写账户 / 资金**，2026-07-15 起 AI 不管账户）。
