# 复盘记录（reviews/）

存放每次交易复盘的报告与配套数据/图。

## 用途与分工

| 目录 | 内容 |
|---|---|
| `signals/` | 信号**事实**记录（开/平仓信号、响铃 log）——复盘的**数据源** |
| `reviews/` | 事后复盘**分析**（统计、逐笔点评、教训、贝叶斯演化）——本目录 |
| `archive/` | 更早的**历史归档**（含旧复盘，如模拟盘复盘、MU 事后复盘） |

`signals/` 只记信号事实、不写分析评价，避免污染复盘数据源；复盘分析放本目录。

## 命名约定

- `YYYY-MM-DD-review.md` —— 复盘主报告（港美混合复盘用此；若港美分开复盘，仿信号文件加 `-HKT`/`-ET`）。
- `YYYY-MM-DD-*.{csv,png}` —— 该次复盘的配套附件（输入数据 CSV、统计图等），文件名带同日日期前缀，与主报告关联。

示例（2026-07-21 全量复盘）：

```
2026-07-21-review.md            # 主报告（16 个 T 全量复盘）
2026-07-21-trades-hk.csv        # 港股 12 个 T 输入数据
2026-07-21-trades-us.csv        # 美股 4 个 T 输入数据
2026-07-21-bayes-evolution.png  # 贝叶斯 P(EV>0) 随样本累加演化图
```

## 复盘规范

- **触发**：复盘由用户指令触发，盘中/盘后 AI 不自主复盘（见 trade skill「复盘分析」）。
- **样本定义（T）**：每次平仓/了结事件 = 一个 T（主动平 🔴、减仓 🟠、移动止损 🟡 被动平均算）。持仓不过夜纪律下不存在未平仓样本。
- **统计口径**：R-multiple 体系（R = P/max_loss），终局统计 + 完整贝叶斯 NIG（t 后验、σ 联合估、带 σ 不确定区间）+ 频率派 CI + 样本量规划，统一由 `.claude/skills/trade/scripts/review.py` 实算，不心算报数。
- **数值结论一律实算、带区间**；小样本（σ 不确定跨度 >5pp）只取方向、不作加仓/改策略决策。
- **盈亏性质**：复盘盈亏按信号参考价与 max_loss 实算，**不涉及真实账户资金**（信号模式：AI 不管账户）。

## 可复现

每次复盘主报告会附当次输入 CSV，配合 `.claude/skills/trade/scripts/review.py` 可重新跑出全部统计；贝叶斯累加图口径与该脚本一致。
